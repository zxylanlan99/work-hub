## 一、设计概述

### 1.1 设计目标

| # | 目标 | 衡量标准 |
|---|------|----------|
| 1 | 极简技术栈，零代码基础可维护 | 纯 HTML/CSS/JS，无框架依赖，8 个原型直接改造为产品代码 |
| 2 | 数据存云端，多设备自动同步 | 所有数据通过 CloudBase 云数据库持久化，任意设备登录即同步 |
| 3 | 核心学习闭环可运行 | 首页→学习计划→知识库→AI对话→复习→沉淀，6 模块完整闭环 |
| 4 | AI API Key 不暴露前端 | 通过 CloudBase 云函数代理，Key 仅存云函数环境变量 |
| 5 | 月度成本控制在 ¥5 以内 | CloudBase 免费额度 + 低成本 AI API |

### 1.2 设计范围

| 类型 | 说明 |
|------|------|
| **包含** | 首页、学习计划（含 5 个 AI 节点）、资讯（含 6 个 AI 节点）、知识库（含 7 个 AI 节点）、AI 对话（含 2 个 AI 节点）、复习计划（含 5 个 AI 节点）、知识沉淀（含 4 个 AI 节点）、系统设置、用户登录（CloudBase Auth 匿名登录） |
| **不包含** | 多租户管理后台、实时协作、PWA 离线模式、移动端 App、向量语义搜索、文件上传、暗色主题 |
| **MVP 阶段（V1）** | 首页 + 学习计划 + 知识库 + AI 对话 + 复习计划 + 笔记（知识沉淀简化版） |
| **后续迭代** | V1.5 资讯+复习完善 → V2 完整知识沉淀+系统设置+输出 → V2.5 多用户+语义搜索 → V3 PWA+移动端 |

### 1.3 设计约束

| 约束类型 | 具体约束 |
|----------|----------|
| **技术约束** | 纯 HTML/CSS/ES6 原生 JS，不引入 React/Vue/TypeScript；CloudBase NoSQL（无 SQL JOIN、无 DB 级外键约束、无存储过程） |
| **资源约束** | CloudBase 免费额度（2GB 存储、5万次读/天、3万次写/天、云函数 1000次/天、静态托管 1GB/5GB月流量） |
| **安全约束** | AI API Key 必须通过云函数代理，前端不可直连；用户数据严格隔离（每集合配置安全规则） |
| **平台约束** | 桌面端浏览器（Chrome/Firefox/Safari/Edge 最新两版），不要求移动端适配 |
| **开发者约束** | 零代码基础，个人本地使用 → 后期可推广 |

### 1.5 术语定义

| 术语 | 说明 |
|------|------|
| CloudBase | 腾讯云一站式后端云服务（BaaS），提供云数据库 + 云函数 + 认证 + 静态托管 |
| 云数据库 | CloudBase NoSQL JSON 文档型数据库，一个集合（Collection）对应一个数据类型，一条文档（Document）对应一条记录 |
| 云函数 | CloudBase 无服务器函数计算，本项目中用于 AI API Key 代理 |
| 安全规则 | CloudBase 数据库 JSON 格式权限控制配置，实现多用户数据隔离 |
| MiMo | 小米大模型平台，主力 AI 模型，兼容 OpenAI API 格式 |
| SiliconFlow | 国内 AI 模型聚合平台，提供 DeepSeek/Kimi 等备选模型统一 API |
| SM-2 | 间隔重复算法，用于复习卡片排期 |

### 2.2 部署架构说明

```
用户浏览器（多台电脑）
    │
    ├─→ CloudBase JS SDK ──→ CloudBase 云数据库
    │       (CRUD 操作)         (NoSQL 文档持久化，广州/上海节点)
    │
    ├─→ CloudBase 云函数 ──→ MiMo / SiliconFlow / 豆包 API ──→ AI 大模型
    │       (AI Key 代理)       (AI 对话/摘要/评分)
    │
    └─→ CloudBase 静态托管 CDN ──→ 静态文件分发
            (HTML/CSS/JS 加载，国内 CDN 加速)
```

- **部署域**：`https://env-id.tcloudbaseapp.com`（默认），支持自定义域名
- **节点选择**：广州/上海（国内低延迟）
- **CDN**：CloudBase 自带国内 CDN，HTML/CSS/JS 分发速度有保障

### 2.3 技术选型

| 层级 | 选型 | 版本/规格 | 理由 |
|------|------|-----------|------|
| 前端框架 | **无框架**，纯 HTML + CSS + JS | ES6 | 零学习成本，8 个原型直接改造 |
| CSS | 纯 CSS，`:root` 变量体系 | — | 原型中已完整定义，无需迁移 |
| JavaScript | 原生 ES6（无 TypeScript） | — | 降低入门门槛 |
| 后端框架 | **无后端** | — | CloudBase JS SDK 直接操作云数据库 |
| 数据库 | **CloudBase 云数据库** | NoSQL 文档型 | 腾讯云托管，多设备同步，免费 2GB |
| SDK | 无 ORM，使用 CloudBase JS SDK | cloudbase-js-sdk@1.8 | `db.collection().get()` 即查即用 |
| 认证 | **CloudBase Auth** | 免费 | 邮箱登录 |
| AI SDK | 无框架，直接 Fetch API（经云函数代理） | — | 减少依赖 |
| AI 模型 | **MiMo**（主力）+ **SiliconFlow**（DeepSeek/Kimi 备选）+ **豆包**（补充） | API | 多模型路由，国内直连 |
| AI Key 代理 | **CloudBase 云函数** | 免费 1000次/天 | Key 存云函数环境变量，前端不暴露 |
| 向量检索 | **暂不支持** | — | V2+ 考虑（可接入外部向量服务） |
| 部署 | **CloudBase 静态网站托管** | 免费 1GB | 同控制台管理，自动 HTTPS |

#### 为什么不用前端框架（React / Vue）

| 原因 | 说明 |
|------|------|
| 你的背景 | 零代码基础，React 学习曲线约 2-3 个月 |
| 原型复用 | 8 个原型是纯 HTML，可直接改造，不需重写 |
| 功能匹配 | 本项目是内容型工具，非重型交互应用，原生 JS 完全够 |
| 渐进升级 | 将来如果需要 React，CloudBase 数据库不变，只换前端 |

#### 为什么选 CloudBase

| 对比维度 | CloudBase | Supabase | 自建后端 |
|----------|-----------|----------|----------|
| 数据库 | NoSQL JSON 文档型 | PostgreSQL（SQL） | 需自己搭建 |
| 免费额度 | 2GB + 5万次读/天 + 3万次写/天 | 500MB + 200万次/月 | 无 |
| API 调用 | JS SDK 直接操作 | 自动 REST API | 需手写 |
| 云函数 | 内置（AI Key 代理） | Edge Functions | 需自建 |
| 国内访问 | 国内节点，低延迟 | 新加坡节点 | — |
| 微信生态 | 天然打通（小程序/公众号） | 不支持 | — |
| 账单整合 | 腾讯云统一账号 | 独立平台 | — |
| 迁移便利 | 标准 MongoDB 协议，可导出 | 开源，标准 PostgreSQL | — |

**选型理由**：用户已有腾讯云账号和运行中的小程序，CloudBase 可实现同一账号下的账单整合与资源统一管理，国内访问延迟远低于 Supabase 新加坡节点。云函数内置支持为 AI Key 安全代理提供了零配置方案。

    ┌──────────┐  ┌──────────┐

**依赖规则**：
- 知识库是唯一中枢，所有模块的数据最终汇聚于知识库或从中读取
- 每个模块通过 CloudBase 云数据库共享数据，模块间不直接调用对方函数
- 用户模块是所有模块的入口前置条件（未登录 → 跳转登录页）

## 四、业务实体模型

### 4.1 实体分类说明

本项目的业务实体按四层分类：

| 层次 | 定义 | 判断标准 | 本项目中数量 |
|------|------|----------|:--:|
| **业务对象**（BO） | 用户可感知的功能领域统称，是实体的聚合边界 | 对应一个独立的用户场景闭环 | 6 |
| **核心实体**（聚合根） | 拥有独立生命周期，可独立创建/修改/删除 | 删除它时级联影响其下的属性实体 | 5 |
| **属性实体**（子实体） | 生命周期依附于核心实体 | 父实体不存在时，自身失去业务意义 | 7 |
| **关系实体**（桥接实体） | 核心价值在于描述两个实体之间的关联 | 存在意义是建立连接 | 1 |

### 4.2 业务对象总览

| 业务对象 | 所属模块 | 聚合根 | 说明 |
|----------|---------|--------|------|
| 学习计划体系 | 学习计划 | 学习目标 | 从目标拆分到里程碑再到每日任务的三层分解 |
| 知识存储体系 | 知识库 | 知识条目 | 对知识条目进行分类、检索、复习的完整闭环 |
| 信息摄入体系 | 资讯 | — | 外部信息流入、AI评分筛选、一键沉淀到知识库 |
| AI 对话体系 | AI 对话 | AI 对话 | 多轮问答、多模型对比、对话结果收藏与沉淀 |
| 知识输出体系 | 知识沉淀 | 输出文档 | 碎片收集→AI辅助整理→结构化输出文档 |
| 系统配置 | 系统设置 | 用户设置 | 每用户唯一一份偏好配置 |

### 4.3 核心实体（5 个聚合根）

| 核心实体 | 所属业务对象 | 独立生命周期体现 | 级联影响 |
|----------|-------------|----------------|---------|
| **学习目标** Goal | 学习计划体系 | 可脱离里程碑独立存在（仅标题+描述即可） | 删目标 → 删全部里程碑 → 删全部任务 |
| **知识条目** Knowledge Item | 知识存储体系 | 可脱离分类/卡片/文档独立存在（仅标题+正文即可） | 删条目 → 卡片失效、关联文档解除引用 |
| **AI 对话** Chat | AI 对话体系 | 可脱离消息独立存在（新建对话无消息） | 删对话 → 级联删全部消息 |
| **输出文档** Output Doc | 知识输出体系 | 可脱离碎片/知识关联独立存在（直接写文档） | 删文档 → 解除碎片/知识条目引用 |
| **用户设置** User Setting | 系统配置 | 用户首次登录时自动创建，全局唯一一份，不可删除 | 无级联 |

### 4.4 属性实体（7 个子实体）

| 属性实体 | 依附的核心实体 | 依附方式 | 生命周期规则 |
|----------|:------------:|---------|-------------|
| **里程碑** Milestone | 学习目标 Goal | `goalId` 引用 | 目标归档/删除 → 级联删除里程碑和任务 |
| **学习任务** Task | 里程碑 Milestone | `milestoneId` 引用 | 里程碑删除 → 级联删除关联任务 |
| **对话消息** Message | AI 对话 Chat | `chatId` 引用 | 对话删除 → 级联删除全部消息 |
| **复习卡片** Review Card | 知识条目 Knowledge Item | `knowledgeId` 引用 | 条目删除 → 卡片标记失效 |
| **复习记录** Review History | 复习卡片 Review Card | `cardId` 引用 | 卡片删除 → 记录归档或删除 |
| **知识分类** Category | 知识条目 Knowledge Item | `categoryId` 引用 | 分类删除 → 条目分类字段置空，条目保留 |
| **灵感碎片** Scrap | 输出文档 Output Doc | `outputDocId` 引用（转化后） | 文档删除 → 碎片解除引用，恢复 raw 状态 |

### 4.5 关系实体（1 个桥接实体）

| 关系实体 | 连接方向 | 关系含义 | 关键属性 |
|----------|---------|---------|---------|
| **资讯条目** News Item | 外部信息源 → 知识条目 | 信息经过阅读/AI评分后，可由用户一键"保存"转化为知识条目，此时 `knowledgeId` 字段建立关联 | `sourceUrl`（来源） + `knowledgeId`（转化目标） |

### 4.6 实体关系图

```
用户 (CloudBase Auth 内置)
  │   （通过 _openid 关联）
  │
  ├── 1:N ── 学习目标 Goal ── 1:N ── 里程碑 Milestone ── 1:N ── 学习任务 Task
  │
  ├── 1:N ── 知识分类 Category ←── N:1 ── 知识条目 Knowledge Item
  │                                            │
  │                                            ├── 1:N ── 复习卡片 Review Card
  │                                            │              │
  │                                            │              └── 1:N ── 复习记录 Review History
  │                                            │
  │                                            └── M:N ── 输出文档 Output Doc
  │                                                          │
  │                                                          └── 1:N ── 灵感碎片 Scrap
  │
  ├── 1:N ── 资讯条目 News Item ──(可转化)→ 知识条目
  │
  ├── 1:N ── AI 对话 Chat ── 1:N ── 对话消息 Message
  │              │
  │              └──(可沉淀)→ 知识条目 / 灵感碎片
  │
  └── 1:1 ── 用户设置 User Setting
```

### 4.7 业务规则

| 规则编号 | 规则描述 | 影响实体 | 实施位置 |
|:--:|----------|----------|----------|
| BR-01 | 删除目标时，应用层级联删除其下所有里程碑和任务 | Goal/Milestone/Task | 应用层 JS |
| BR-02 | 删除对话时，应用层级联删除所有关联消息 | Chat/Message | 应用层 JS |
| BR-03 | 删除知识条目时，关联复习卡片标记为无效（不清除历史记录） | Knowledge Item/Review Card | 应用层 JS |
| BR-04 | 删除分类时，该分类下的知识条目 `categoryId` 置空，条目本身保留 | Category/Knowledge Item | 应用层 JS |
| BR-05 | 每个用户只能有一份 `user_settings`（_openid 唯一） | User Setting | 应用层 JS |
| BR-06 | 知识条目软删除（`isDeleted: true`），30 天后可永久清理 | Knowledge Item | 应用层 JS |
| BR-07 | 资讯条目 24 小时内同一 URL 不允许重复提交 | News Item | 应用层 JS |
| BR-08 | 复习卡片由 AI 生成，人工确认后写入（不可重复出同一张卡片） | Review Card | 应用层 JS |
| BR-09 | 灵感碎片转化后 `status: converted`，关联 `outputDocId` | Scrap | 应用层 JS |

> **注意**：由于 CloudBase NoSQL 不支持数据库级外键约束和级联删除，以上所有业务规则均由应用层 JavaScript 代码保证。

（完整字段定义见详细设计的逻辑数据模型章节。此处列核心属性以便 ER 理解。）

| 资讯条目 News Item | 标题、来源URL、来源名称、分类、标签、AI评分、已读/已保存/关联知识ID |

## 五、接口设计

### 5.1 总体说明

本项目的"接口"包含三类调用契约：

| 类别 | 调用方式 | 接口形式 | 说明 |
|------|----------|----------|------|
| 数据库操作 | CloudBase JS SDK | `db.collection('xxx').xxx()` | 核心 CRUD，前端直连云数据库 |
| AI 调用 | CloudBase 云函数代理 | `app.callFunction('ai-proxy', data)` | AI Key 安全调用 |
| 认证 | CloudBase Auth SDK | `auth.anonymousAuthProvider().signIn()` | 匿名登录/登出/状态查询（V2 支持微信登录） |

### 5.2 数据库操作接口清单

13 个集合，每集合 4 种操作（增/删/改/查）+ 条件查询：

| 集合 | SDK 调用路径 | 支持的操作 |
|------|-------------|-----------|
| `goals` | `db.collection('goals')` | `.add()` `.get()` `.doc().update()` `.doc().remove()` |
| `milestones` | `db.collection('milestones')` | 同上 |
| `tasks` | `db.collection('tasks')` | 同上 |
| `categories` | `db.collection('categories')` | 同上 |
| `knowledge_items` | `db.collection('knowledge_items')` | 同上 |
| `news_items` | `db.collection('news_items')` | 同上 |
| `chats` | `db.collection('chats')` | 同上 |
| `messages` | `db.collection('messages')` | 同上 |
| `review_cards` | `db.collection('review_cards')` | 同上 |
| `review_history` | `db.collection('review_history')` | 同上 |
| `output_docs` | `db.collection('output_docs')` | 同上 |
| `scraps` | `db.collection('scraps')` | 同上 |
| `user_settings` | `db.collection('user_settings')` | 同上 |

**典型查询示例**：

```javascript
// 获取进行中的学习目标
const { data } = await db
  .collection('goals')
  .where({ _openid: uid, status: 'active' })
  .orderBy('createdAt', 'desc')
  .get()

// 搜索知识条目（关键词模糊匹配）
const { data } = await db
  .collection('knowledge_items')
  .where({
    _openid: uid,
    title: db.RegExp({ regexp: keyword, options: 'i' })
  })
  .get()

// 获取今日待复习卡片
const { data } = await db
  .collection('review_cards')
  .where({
    _openid: uid,
    nextReview: db.command.lte(new Date())
  })
  .orderBy('nextReview', 'asc')
  .get()
```

### 5.3 AI 调用接口（云函数 ai-proxy）

**入参**（前端 → 云函数）：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `messages` | `Array` | 是 | 消息数组，格式 `[{role, content}]` |
| `model` | `string` | 是 | 模型代号：`mimo` / `deepseek` / `kimi` / `doubao` / `deepseek-r1` |
| `stream` | `boolean` | 否 | 默认 `false`（MVP 暂不启用流式） |

**出参**（云函数 → 前端）：

```javascript
// 成功
{
  content: "AI 回复内容",
  tokens: 245,
  model: "mimo-v2.5-pro"
}

// 失败
{
  error: "RATE_LIMITED" | "TIMEOUT" | "API_ERROR",
  message: "请求过于频繁，请稍后再试" | "AI 响应超时" | "HTTP 429"
}
```

**前端调用示例**：

```javascript
// 单模型调用
const result = await app.callFunction({
  name: 'ai-proxy',
  data: {
    messages: [
      { role: 'system', content: '你是一个学习助手' },
      { role: 'user', content: '帮我解释什么是闭包' }
    ],
    model: 'mimo'
  }
})

// 双模型对比调用（前端并发两个 callFunction）
const [result1, result2] = await Promise.all([
  app.callFunction({ name: 'ai-proxy', data: { messages, model: 'mimo' } }),
  app.callFunction({ name: 'ai-proxy', data: { messages, model: 'kimi' } })
])
```

### 5.4 认证接口

| 操作 | SDK 调用 | 说明 |
|------|----------|------|
| 注册/登录 | `auth.signInWithEmailAndPassword(email, password)` | CloudBase 邮箱登录自动处理首次注册 |
| 获取当前用户 | `auth.getLoginState()` | 返回值 `.user.uid` 为用户唯一标识 |
| 登出 | `auth.signOut()` | 清除本地登录态 |
| 密码重置 | `auth.sendPasswordResetEmail(email)` | 发送重置邮件 |
| 匿名登录 | `auth.signInAnonymously()` | V1.5+ 可选 |

### 5.5 接口规范约定

| 规范项 | 约定 |
|--------|------|
| 用户标识 | 所有数据操作自动附加 `_openid`（CloudBase Auth 注入，前端不传） |
| 时间戳 | 写入使用 `db.serverDate()`，读取返回 ISO 字符串 |
| 分页 | 使用 `.skip(N)` + `.limit(N)`，单次最多 100 条 |
| 排序 | `.orderBy('field', 'asc'/'desc')`，可链式多次调用 |
| 错误处理 | 统一检查 `result.code || catch(err)`，具体错误码见第九章 |
| 安全规则 | 所有集合配置 `{"read": "auth.openid == doc._openid", "write": "auth.openid == doc._openid"}` |

## 六、数据设计

### 6.1 数据流向全景

```
用户操作 → 前端 JS → CloudBase JS SDK → CloudBase 云数据库 → 数据持久化
                │
                ├─→ CloudBase 云函数 (ai-proxy) → MiMo / SiliconFlow / 豆包 API → AI 大模型
                │       ↑
                │   (API Key 环境变量)
                │
                └─→ CloudBase 静态托管 CDN ← 静态文件（HTML/CSS/JS）
```

### 6.2 数据存储策略

| 存储位置 | 存储内容 | 策略说明 |
|----------|----------|----------|
| **CloudBase 云数据库** | 全部业务数据（学习目标/里程碑/任务/知识条目/分类/资讯/对话/消息/复习卡片/复习记录/输出文档/灵感碎片/用户设置） | 唯一持久化存储，多设备云同步 |
| **浏览器 localStorage** | 最近一次的 UI 状态（侧边栏折叠、当前 Tab、字体大小偏好） | 仅 UI 便捷状态，清除后不影响数据完整性 |
| **云函数环境变量** | AI API Key（MiMo / SiliconFlow / 豆包） | 云端安全存储，前端不可接触 |
| **前端全局变量** | CloudBase 环境 ID（写在 `js/cloudbase.js`） | 非敏感信息 |

## 七、安全设计

### 7.1 认证方案

| 项目 | 方案 |
|------|------|
| 认证服务 | CloudBase Auth（邮箱登录） |
| 用户唯一标识 | `loginState.user.uid`（CloudBase 自动管理） |
| 登录流程 | `auth.signInWithEmailAndPassword(email, pwd)` → 获取 `loginState` → 存储 `uid` 到全局变量 |
| 路由守卫 | 每个页面加载时 `js/auth.js` 调用 `auth.getLoginState()`，未登录 → 跳转 `login.html` |
| 会话保持 | CloudBase Auth 自动持久化，`persistence: 'local'` |

### 7.2 授权方案（安全规则）

CloudBase 安全规则基于 JSON 配置，每个集合独立设置：

```json
{
  "read": "auth.openid == doc._openid",
  "write": "auth.openid == doc._openid"
}
```

- `auth.openid`：CloudBase Auth 在用户登录后自动注入请求上下文的用户标识
- `doc._openid`：待操作文档中的用户标识字段
- 规则含义：用户只能读写自己创建的文档
- 未登录用户：所有集合的读写操作被拒绝

**特殊规则**：`user_settings` 额外应用层约束——每个用户只能有一条设置文档。

### 7.3 API Key 管理

| 环境 | 方案 | 风险等级 |
|------|------|:--:|
| **MVP** | AI Key 存 CloudBase 云函数环境变量（`AI_SILICONFLOW_KEY`、`AI_DOUBAO_KEY`），前端通过 `app.callFunction('ai-proxy')` 调用 | 低 |
| **推广后** | 继续使用云函数代理，可按需升级为腾讯云 API 网关（支持限流、鉴权、监控） | 低 |
| **生产** | 云函数 + API 网关（按量计费，支持 IP 白名单） | 低 |

> 整个方案中，AI Key 从不经过浏览器网络传输，即使前端代码被完全查看也无法获取 Key。

### 7.4 数据加密策略

| 层级 | 加密方式 | 说明 |
|------|----------|------|
| 传输层 | HTTPS（CloudBase 自动） | SDK 通信、静态托管访问均走 TLS |
| 存储层 | CloudBase 云端自动加密 | 腾讯云数据库默认加密 |
| 应用层 | MVP 不额外加密，V1.5+ 考虑知识条目的 AES 加密 | `level: 'secret'` 字段预留 |

## 八、性能设计

### 8.1 性能指标

| 指标 | MVP 目标 | 说明 |
|------|:--:|------|
| 首页加载时间 | ≤ 2s（首次）/ ≤ 1s（缓存） | CDN 静态文件 + CloudBase 数据查询 |
| 页面切换时间 | ≤ 500ms | 独立 HTML 文件，浏览器缓存 |
| 数据库单次查询响应 | ≤ 500ms | CloudBase NoSQL 国内节点 |
| AI 单次响应时间 | ≤ 10s | 经云函数代理，MiMo API 国内直连 |
| 云函数冷启动 | ≤ 3s | CloudBase 云函数 256MB 内存 |
| 并发用户 | 1（个人使用） | MVP 阶段无并发压力 |

### 8.2 缓存策略

| 缓存层级 | 内容 | 失效策略 | 实现方式 |
|----------|------|----------|----------|
| **浏览器 HTTP 缓存** | 静态文件（CSS/JS） | `Cache-Control: max-age=3600`（CloudBase 静态托管默认） | 自动 |
| **CloudBase JS SDK 缓存** | 用户登录态 | 7 天（Auth SDK 默认） | 自动 |
| **应用层内存缓存** | 当前页面的数据列表 | 页面跳转时清除 | JS 变量暂存 |
| **localStorage** | UI 偏好（主题、字体） | 手动清除时失效 | `localStorage.setItem()` |

> MVP 阶段不引入 Service Worker 或 IndexedDB 离线缓存。V1.5+ 评估 PWA 离线方案。

### 8.3 优化方案

| 优化点 | 方案 | 实施阶段 |
|--------|------|:--:|
| **静态资源优化** | CSS/JS 文件已按模块拆分，每个 HTML 只加载自己需要的 JS（如 `plan.html` 只加载 `plan.js`） | MVP |
| **SDK 按需加载** | CloudBase JS SDK CDN 异步加载（`<script async>`） | MVP |
| **数据库查询优化** | 使用复合索引（`_openid + status` 等），避免全表扫描 | MVP |
| **云函数预热** | CloudBase 云函数默认无冷启动优化，256MB 内存足够简单代理调用 | MVP |
| **AI 并发控制** | 前端限制同时发起的 AI 请求数 ≤ 2 | MVP |
| **图片优化** | 本地图标使用 SVG，避免外部图片依赖 | MVP |

### 8.4 性能监控

| 监控方式 | 指标 | 工具 |
|----------|------|------|
| 浏览器 DevTools | 页面加载时间、网络请求耗时 | Chrome Performance |
| CloudBase 控制台 | 数据库读写次数、云函数调用次数/耗时 | 腾讯云监控 |

## 九、错误处理

### 9.1 错误分类

| 错误类型 | 说明 | 示例 | 处理层级 |
|----------|------|------|:--:|
| **网络错误** | 用户断网、CDN 不可达、DNS 故障 | 打开页面白屏、数据加载失败 | 前端兜底 |
| **认证错误** | 登录态过期、Token 无效、密码错误 | `auth.signIn` 返回错误 | 前端提示 |
| **权限错误** | 安全规则拒绝访问 | `DATABASE_PERMISSION_DENIED` | 前端提示 + 检查 _openid |
| **数据错误** | 数据不存在、字段校验失败、重复提交 | 空集返回、插入失败 | 前端提示 + 数据校验 |
| **AI 错误** | AI API 超时、限流、Key 失效、返回格式异常 | `RATE_LIMITED`、`TIMEOUT` | 前端提示 + 降级 |
| **系统错误** | 云函数崩溃、CloudBase 服务异常 | 500 错误、函数执行超时 | 前端提示 + 重试 |
| **业务错误** | 违反业务规则 | 24h 内重复提交同一 URL | 前端拦截 |

### 9.2 错误码定义

| 错误码 | 类型 | 说明 | 处理建议 |
|--------|------|------|----------|
| `NETWORK_ERROR` | 网络错误 | 网络连接失败 | 提示"网络连接失败，请检查网络后重试"，可重试 |
| `AUTH_NOT_LOGGED_IN` | 认证错误 | 未登录或登录态过期 | 跳转登录页 |
| `AUTH_INVALID_CREDENTIALS` | 认证错误 | 邮箱或密码错误 | 提示"邮箱或密码错误" |
| `AUTH_EMAIL_EXISTS` | 认证错误 | 邮箱已注册 | 提示"该邮箱已注册，请直接登录" |
| `DB_PERMISSION_DENIED` | 权限错误 | 安全规则拒绝 | 提示"无权限操作"，检查登录态 |
| `DB_DOC_NOT_FOUND` | 数据错误 | 文档不存在 | 提示"数据不存在或已被删除" |
| `DB_WRITE_FAILED` | 数据错误 | 写入失败 | 提示"保存失败，请重试" |
| `DB_DUPLICATE` | 业务错误 | 数据重复 | 提示"该内容已存在" |
| `AI_RATE_LIMITED` | AI 错误 | API 请求过于频繁 | 提示"请求过于频繁，请稍后再试" |
| `AI_TIMEOUT` | AI 错误 | AI 响应超时（>10s） | 提示"AI 响应超时，请重试" |
| `AI_API_ERROR` | AI 错误 | AI 服务返回异常 | 提示"AI 服务暂时不可用"，可重试 |
| `AI_KEY_INVALID` | 系统错误 | API Key 无效（云函数配置错误） | 提示"服务配置异常，请联系管理员" |
| `AI_PARSE_ERROR` | AI 错误 | AI 返回内容无法解析 | 提示"AI 返回格式异常"，显示原始内容兜底 |
| `SCF_TIMEOUT` | 系统错误 | 云函数执行超时 | 提示"服务繁忙，请重试" |
| `VALIDATION_ERROR` | 业务错误 | 前端数据校验失败 | 提示具体校验规则（如"标题不能为空"） |
| `BUSINESS_RULE_ERROR` | 业务错误 | 违反业务规则 | 提示具体规则说明（如"过去24小时内已提交该文章"） |

### 9.3 错误提示规范

| 规范项 | 说明 |
|--------|------|
| 展示位置 | 使用 `utils.js` 中的 `showToast(msg, type)` 统一提示，type 为 `error` / `warning` / `info` |
| 文案要求 | 中文提示，说明"发生了什么 + 建议怎么做"，避免暴露技术细节 |
| 持续时间 | 普通错误 3s 自动消失，重要错误（如登录失败）需用户手动关闭 |
| 降级兜底 | AI 调用失败时，前端保留用户输入的消息，允许重试 |
| 重试机制 | 网络错误、AI 超时支持一键重试（复用相同参数重新调用） |
| 关键操作 | 保存/删除类操作失败时，保留用户已输入内容，防止数据丢失 |

### 9.4 日志策略

| 日志类型 | 记录内容 | 存储位置 | 用途 |
|----------|----------|----------|------|
| **开发调试日志** | 所有 SDK 调用入参/出参、错误堆栈 | `console.log` / `console.error` | 开发阶段排查问题 |
| **AI 调用日志** | 模型、tokens、耗时、是否成功 | 云函数 `console.log` → CloudBase 日志 | 成本追踪 |
| **关键操作日志** | 创建/删除目标、知识入库、卡片复习 | `console.log`（MVP）→ V1.5+ 存集合 `operation_logs` | 审计追溯 |
| **错误日志** | 所有 catch 到的错误 | `console.error`（MVP）→ V1.5+ 接入腾讯云日志服务 | 运维监控 |

> MVP 阶段所有日志通过 `console.log/error` 输出，在浏览器 DevTools 中查看。生产阶段可升级为 CloudBase 云函数日志服务。

## 十、部署方案

### 10.1 部署环境

| 环境 | 域名 | CloudBase 环境 | 说明 |
|------|------|----------------|------|
| **开发/生产** | `https://env-id.tcloudbaseapp.com` | 同一环境 | MVP 阶段开发和生产力不分离，个人使用无需多环境 |

### 10.2 部署流程

用户已有腾讯云账号和环境。在同账号下新建 CloudBase 环境即可：

```
1. 登录腾讯云控制台 → CloudBase → 新建环境（按量计费，免费额度个人用远够）
2. 在 CloudBase 控制台开通：
   - 云数据库（NoSQL 文档型）
   - 云函数
   - 静态网站托管
   - 登录认证（邮箱方式）
3. 创建 13 个集合（goals/milestones/tasks/categories/knowledge_items/
   news_items/chats/messages/review_cards/review_history/output_docs/
   scraps/user_settings），配置安全规则
4. 创建 ai-proxy 云函数，配置环境变量（AI_SILICONFLOW_KEY, AI_DOUBAO_KEY）
5. 安装 CloudBase CLI：npm i -g @cloudbase/cli
6. 本地执行 cloudbase login → 选择对应环境
7. 执行 cloudbase hosting deploy ./ -e your-env-id
8. 得到 https://your-env-id.tcloudbaseapp.com 域名
```

AI_SILICONFLOW_KEY=sk-xxxxx
