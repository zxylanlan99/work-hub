## 一、设计约定

### 1.1 命名规范

| 层级 | 规范 | 示例 |
|------|------|------|
| 集合名 | `snake_case`，复数 | `knowledge_items`、`review_cards` |
| 字段名 | `camelCase` | `sourceUrl`、`isDeleted`、`createdAt` |
| 索引名 | `idx_<collection>_<field1>_<field2>` | `idx_review_cards_nextReview` |
| 系统字段 | CloudBase 自动注入，应用层不写入 | `_id`、`_openid` |

### 1.2 数据类型映射

| CloudBase NoSQL 类型 | JavaScript 类型 | 说明 |
|---------------------|----------------|------|
| `string` | `string` | 文本，可指定最大长度约束 |
| `number` | `number` | 整数或浮点数 |
| `boolean` | `boolean` | true / false |
| `Date` | `Date` / `serverDate()` | 时间戳，用 `serverDate()` 自动写入 |
| `Array<string>` | `Array` | 字符串数组（如标签、关联 ID 列表） |
| `Object` | `Object` | 嵌套对象（本设计未使用，避免深层嵌套查询困难）|
| `enum` | `string` | 枚举值，应用层校验 |

### 1.3 系统字段说明

所有集合自动包含：

| 字段 | 类型 | 写入方 | 说明 |
|------|------|--------|------|
| `_id` | string | CloudBase | 主键，集合内全局唯一，只读 |
| `_openid` | string | Auth 登录后自动注入 | 用户标识，**所有查询必须携带** |
| `createdAt` | Date | `serverDate()` | 创建时间，只读 |
| `updatedAt` | Date | `serverDate()` | 更新时间，每次写操作前刷新 |

### 1.4 通用约束

- **软删除规范**：知识类条目使用 `isDeleted: boolean` 标记删除，`deletedAt` 记录删除时间，30 天后由定时云函数永久清理
- **级联删除**：CloudBase NoSQL 无 DB 级外键，所有级联由应用层 JS 保证（详见 §四 关联映射）
- **唯一性约束**：应用层校验（如 `news_items.sourceUrl` 同用户不重复），CloudBase 无 UNIQUE 约束

## 三、索引汇总与 DDL

### 3.1 索引设计原则

1. **所有索引必须以 `_openid` 为首字段**（CloudBase 安全规则要求，且实现多用户数据隔离）
2. **复合索引字段排序**：等值查询字段在前，范围查询字段（`createdAt`、`nextReview`）在后
3. **避免过度索引**：CloudBase 免费版每集合最多 20 个索引

### 3.2 完整索引清单（21 个）

| # | 集合 | 索引组合 | 排序 | 查询场景 |
|:--:|------|---------|:----:|---------|
| 1 | `goals` | `_openid + status` | ASC/ASC | 首页：加载进行中目标 |
| 2 | `goals` | `createdAt` | DESC | 目标列表按创建时间倒序 |
| 3 | `milestones` | `goalId + sort` | ASC/ASC | 目标详情：加载里程碑列表 |
| 4 | `tasks` | `milestoneId + sort` | ASC/ASC | 里程碑详情：加载任务列表 |
| 5 | `tasks` | `goalId + status` | ASC/ASC | 目标总进度统计 |
| 6 | `knowledge_items` | `_openid + isDeleted` | ASC/ASC | 主列表：排除已删除 |
| 7 | `knowledge_items` | `categoryId` | ASC | 按分类筛选 |
| 8 | `knowledge_items` | `createdAt` | DESC | 最新知识 |
| 9 | `knowledge_items` | `tags` | — | 标签检索（无索引，前端过滤） |
| 10 | `review_cards` | `_openid + nextReview + mastery` | ASC/ASC/ASC | ⭐ 今日待复习队列（最关键） |
| 11 | `review_cards` | `knowledgeId` | ASC | 某知识的复习卡片 |
| 12 | `review_history` | `cardId + reviewedAt` | ASC/DESC | 某卡片的复习历史 |
| 13 | `news_items` | `_openid + hasRead` | ASC/ASC | Tab 切换：未读/已读 |
| 14 | `news_items` | `createdAt` | DESC | 最新资讯 |
| 15 | `chats` | `createdAt` | DESC | 对话列表按时间倒序 |
| 16 | `messages` | `chatId + createdAt` | ASC/ASC | ⭐ 对话历史按时间正序 |
| 17 | `output_docs` | `_openid + status` | ASC/ASC | 输出列表：草稿/已发布 |
| 18 | `output_docs` | `createdAt` | DESC | 最新输出 |
| 19 | `scraps` | `outputDocId` | ASC | 某文档的碎片列表 |
| 20 | `scraps` | `createdAt` | DESC | 最新碎片 |
| 21 | `user_settings` | `_openid`（唯一） | — | 读取用户设置（每用户一份） |

### 3.3 索引创建 DDL（CloudBase 控制台 / CLI）

```javascript
// ============ 索引 1-2：goals ============
// 1. 按状态筛选 + 用户隔离
{ "fields": [{ "name": "_openid", "direction": "asc" }, { "name": "status", "direction": "asc" }] }
// 2. 按创建时间倒序
{ "fields": [{ "name": "createdAt", "direction": "desc" }] }

// ============ 索引 3-5：milestones / tasks ============
// 3. milestones: goalId + sort
{ "fields": [{ "name": "goalId", "direction": "asc" }, { "name": "sort", "direction": "asc" }] }
// 4. tasks: milestoneId + sort
{ "fields": [{ "name": "milestoneId", "direction": "asc" }, { "name": "sort", "direction": "asc" }] }
// 5. tasks: goalId + status（目标总进度统计用）
{ "fields": [{ "name": "goalId", "direction": "asc" }, { "name": "status", "direction": "asc" }] }

// ============ 索引 6-8：knowledge_items ============
// 6. 主列表排除已删除
{ "fields": [{ "name": "_openid", "direction": "asc" }, { "name": "isDeleted", "direction": "asc" }] }
// 7. 按分类筛选
{ "fields": [{ "name": "categoryId", "direction": "asc" }] }
// 8. 最新知识
{ "fields": [{ "name": "createdAt", "direction": "desc" }] }

// ============ 索引 10-11：review_cards（最关键）============
// 10. ⭐ 今日待复习队列（必须 _openid 首字段）
{
  "fields": [
    { "name": "_openid", "direction": "asc" },
    { "name": "nextReview", "direction": "asc" },   // 范围查询：<= today
    { "name": "mastery", "direction": "asc" }        // 可选：优先复习低掌握度
  ]
}
// 11. 某知识的复习卡片
{ "fields": [{ "name": "knowledgeId", "direction": "asc" }] }

// ============ 索引 12：review_history ============
// 12. 某卡片的复习历史（按时间倒序，看最近记录）
{
  "fields": [
    { "name": "cardId", "direction": "asc" },
    { "name": "reviewedAt", "direction": "desc" }
  ]
}

// ============ 索引 13-14：news_items ============
// 13. 已读/未读 Tab 切换
{ "fields": [{ "name": "_openid", "direction": "asc" }, { "name": "hasRead", "direction": "asc" }] }
// 14. 最新资讯
{ "fields": [{ "name": "createdAt", "direction": "desc" }] }

// ============ 索引 15：chats ============
// 15. 对话列表按时间倒序（最近对话在前）
{ "fields": [{ "name": "createdAt", "direction": "desc" }] }

// ============ 索引 16：messages（关键）============
// 16. ⭐ 对话历史按时间正序（还原对话流）
{ "fields": [{ "name": "chatId", "direction": "asc" }, { "name": "createdAt", "direction": "asc" }] }

// ============ 索引 17-18：output_docs ============
// 17. 输出列表：草稿/已发布
{ "fields": [{ "name": "_openid", "direction": "asc" }, { "name": "status", "direction": "asc" }] }
// 18. 最新输出
{ "fields": [{ "name": "createdAt", "direction": "desc" }] }

// ============ 索引 19-20：scraps ============
// 19. 某文档的碎片
{ "fields": [{ "name": "outputDocId", "direction": "asc" }] }
// 20. 最新碎片
{ "fields": [{ "name": "createdAt", "direction": "desc" }] }

// ============ 索引 21：user_settings ============
// 21. 每用户唯一（_openid 唯一，无需额外索引，主键查询）
// 注：user_settings 直接用 `_openid` 查询，CloudBase 自动走主键索引
```

## 四、集合关联映射

### 4.1 关联总图

```
用户（_openid）
  │
  ├── 1:N ── goals ── 1:N ── milestones ── 1:N ── tasks
  │            │                        │
  │            │                        └── goalId（冗余，跨里程碑汇总用）
  │            └── _id ← milestoneId
  │
  ├── 1:N ── categories（自引用 parentId）
  │            │
  │            └── _id ← categoryId ──┐
  │                                   │
  ├── 1:N ── knowledge_items ←────────┘
  │            │
  │            ├── 1:N ── review_cards
  │            │            │
  │            │            └── 1:N ── review_history
  │            │
  │            └── M:N ── output_docs（通过 knowledgeIds 数组）
  │
  ├── 1:N ── news_items ──（转化后）──→ knowledge_items._id（knowledgeId 字段）
  │
  ├── 1:N ── chats ── 1:N ── messages
  │              │
  │              └── knowledgeId ──→ knowledge_items._id（可选关联）
  │
  └── 1:1 ── user_settings
```

### 4.2 关联类型说明

| 关联 | 类型 | 实现方式 | 级联规则 |
|------|:----:|---------|---------|
| 用户 → 所有集合 | 1:N | `_openid` 系统字段 | 用户删账号 → 所有数据自动删除（CloudBase Auth 级联） |
| goals → milestones | 1:N | `milestone.goalId` | 删 goal → 应用层级联删 milestones + tasks |
| milestones → tasks | 1:N | `task.milestoneId` | 删 milestone → 应用层级联删 tasks |
| categories 自引用 | 1:N | `category.parentId` | 删父分类 → 子分类 `parentId` 置空，条目保留 |
| knowledge_items → review_cards | 1:N | `card.knowledgeId` | 删条目 → 卡片标记失效（不清除历史） |
| review_cards → review_history | 1:N | `history.cardId` | 删卡片 → 记录保留（归档） |
| output_docs ↔ knowledge_items | M:N | `doc.knowledgeIds` 数组 | 删知识 → 应用层移除数组中的 ID |
| news_items → knowledge_items | 1:1（可选） | `news.knowledgeId` | 删资讯不影响知识，删知识 → `knowledgeId` 置空 |
| chats → messages | 1:N | `msg.chatId` | 删对话 → 应用层级联删消息 |
| chats → knowledge_items | N:1（可选） | `chat.knowledgeId` | 删知识不影响对话 |

### 4.3 应用层级联删除代码示例

```javascript
// 删除学习目标时，级联删除里程碑和任务
async function deleteGoal(goalId) {
  const db = app.database()
  
  // 1. 查找该目标下所有里程碑
  const milestones = await db.collection('milestones')
    .where({ goalId, _openid: uid }).get()
  
  // 2. 逐个删除里程碑（触发其下的任务删除）
  for (const m of milestones.data) {
    await deleteMilestone(m._id)  // 递归删除 tasks
  }
  
  // 3. 删除目标本身
  await db.collection('goals').doc(goalId).remove()
}

async function deleteMilestone(milestoneId) {
  const db = app.database()
  // 删除该里程碑下所有任务
  await db.collection('tasks')
    .where({ milestoneId, _openid: uid }).remove()
  // 删除里程碑
  await db.collection('milestones').doc(milestoneId).remove()
}
```

## 五、数据生命周期

### 5.1 软删除与清理策略

| 集合 | 软删除字段 | 清理时机 | 清理方式 |
|------|----------|---------|---------|
| `knowledge_items` | `isDeleted` / `deletedAt` | 软删除后 30 天 | 定时云函数 `data-cleanup` 永久删除 |
| `review_cards` | 无（硬删除） | 知识条目删除时立即标记 `mastery: 0`（失效）| 应用层过滤 `nextReview` 时不查失效卡片 |
| `news_items` | `isSaved` | 转化后保留 90 天 | 定时云函数归档到"已处理" Tab |
| `messages` | 无 | 对话删除时级联删除 | 应用层 |

### 5.2 定时清理云函数（伪代码）

```javascript
// 云函数：data-cleanup（每天凌晨 2:00 触发）
const cloud = require('@cloudbase/node-sdk')
const app = cloud.init({ env: cloud.SYMBOL_CURRENT_ENV })
const db = app.database()

exports.main = async () => {
  const now = new Date()
  const thirtyDaysAgo = new Date(now - 30 * 24 * 3600 * 1000)
  
  // 1. 清理软删除超过 30 天的知识条目
  const expired = await db.collection('knowledge_items')
    .where({
      isDeleted: true,
      deletedAt: db.command.lt(thirtyDaysAgo)
    }).get()
  
  for (const item of expired.data) {
    await db.collection('knowledge_items').doc(item._id).remove()
    // 同时清理关联的复习卡片
    await db.collection('review_cards')
      .where({ knowledgeId: item._id }).remove()
  }
  
  // 2. 归档 90 天前的已转化资讯
  const ninetyDaysAgo = new Date(now - 90 * 24 * 3600 * 1000)
  await db.collection('news_items')
    .where({ isSaved: true, createdAt: db.command.lt(ninetyDaysAgo) })
    .update({ archived: true })
  
  return { cleaned: expired.data.length }
}
```
