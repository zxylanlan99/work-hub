## 一、接口体系概述

### 1.1 三层接口模型

本架构无传统 RESTful 接口，交互分为三层：

```
┌─────────────────────────────────────────────────────┐
│                  前端（浏览器）                         │
│                                                        │
│  js/auth.js  →  CloudBase Auth SDK  →  用户登录        │
│  js/db.js    →  CloudBase JS SDK   →  云数据库 CRUD    │
│  js/ai-chat.js → app.callFunction() → 云函数 ai-proxy  │
└──────────────────────┬────────────────────────────────┘
                       │ HTTPS（CloudBase 托管域名）
┌──────────────────────▼────────────────────────────────┐
│              CloudBase 云端                             │
│                                                         │
│  云数据库（NoSQL）← 安全规则（_openid 隔离）              │
│  云函数 ai-proxy ← 环境变量（AI API Key 不暴露前端）       │
│  CloudBase Auth ← 匿名登录 / 微信登录                    │
└─────────────────────────────────────────────────────────┘
```

### 1.2 与旧 RESTful 架构的差异

| 维度 | 旧架构（已废弃） | 新架构（本文档） |
|------|----------------|----------------|
| 接口形式 | 33 个 RESTful `/api/*` | CloudBase SDK 直连 + 2 云函数 |
| 认证方式 | 前端 JWT 管理 | CloudBase Auth（_openid 自动注入） |
| AI 调用 | 前端直连 API Key | 云函数 `ai-proxy` 代理 |
| 安全规则 | 后端中间件校验 | CloudBase 数据库安全规则（JSON） |
| 跨域 | CORS 配置 | 无（同域 CloudBase 托管） |

### 1.3 请求流程总图

```
用户操作
  → CloudBase Auth 登录（获取 _openid）
  → CloudBase JS SDK 发起数据库请求（自动携带 _openid）
  → CloudBase 安全规则校验（拒绝跨用户访问）
  → 数据库操作执行
  → 返回结果（前端 then/catch 处理）
```

## 二、CloudBase SDK 数据操作接口

> **前置条件**：前端需引入 CloudBase JS SDK 并完成初始化（见 §四）。

### 2.1 通用调用模式

```javascript
// 初始化（js/db.js）
import cloudbase from '@cloudbase/js-sdk'

const app = cloudbase.init({
  env: 'your-env-id',       // CloudBase 环境 ID
  region: 'ap-shanghai'     // 区域（上海）
})

// 登录后，所有数据库操作自动携带 _openid
const db = app.database()

// 通用查询模板
async function queryCollection(collectionName, whereClause, orderClause) {
  try {
    const { data } = await db
      .collection(collectionName)
      .where(whereClause)
      .orderBy(orderClause.field, orderClause.direction)
      .get()
    return { success: true, data }
  } catch (err) {
    return { success: false, error: err }
  }
}
```

#### 2.2.1 `goals` — 学习目标

| 操作 | SDK 方法 | 示例 |
|------|---------|------|
| **创建** | `.add(doc)` | `db.collection('goals').add({ title, status: 'active' })` |
| **查询（进行中）** | `.where().get()` | `db.collection('goals').where({ status: 'active' }).orderBy('createdAt', 'desc').get()` |
| **查询（单条）** | `.doc(id).get()` | `db.collection('goals').doc(goalId).get()` |
| **更新** | `.doc(id).update(partial)` | `db.collection('goals').doc(goalId).update({ status: 'completed' })` |
| **删除** | `.doc(id).remove()` | `db.collection('goals').doc(goalId).remove()` → 触发应用层级联删除 |

**级联删除应用层实现**（见 `10-物理数据模型设计-v1.0.md` §4.3）

#### 2.2.2 `milestones` — 里程碑

|------|---------|----------------|
| 查某目标下全部 | `.where({ goalId })` | 配合 `.orderBy('sort', 'asc')` |
| 创建 | `.add({ goalId, title })` | `goalId` 必填 |
| 更新排序 | `.doc(id).update({ sort })` | 拖拽后批量更新 |
| 删除 | `.doc(id).remove()` | 触发应用层批量删除 tasks |

#### 2.2.3 `tasks` — 学习任务

| 操作 | SDK 方法 | 说明 |
|------|---------|------|
| 查某里程碑下 | `.where({ milestoneId })` | 按 `sort` 排序 |
| 查某目标下（跨里程碑） | `.where({ goalId })` | 用于目标总进度统计 |
| 更新状态 | `.doc(id).update({ status: 'completed', updatedAt: db.serverDate() })` | 勾选完成 |
| 批量更新 | 逐条 `.update()` | CloudBase 无批量 update，需循环 |

#### 2.2.4 `knowledge_items` — 知识条目

| 操作 | SDK 方法 | 关键说明 |
|------|---------|---------|
| **主列表查询** | `.where({ isDeleted: false })` | 必须排除软删除 |
| 按分类筛选 | `.where({ categoryId })` | 配合 `isDeleted: false` |
| 搜索（标题/内容） | `.where({ title: db.RegExp(...) })` | CloudBase 支持正则，但不支持全文检索 |
| 创建 | `.add({ title, content, ... })` | `createdAt/updatedAt` 自动 `serverDate()` |
| 软删除 | `.doc(id).update({ isDeleted: true, deletedAt: db.serverDate() })` | **不物理删除** |
| 永久删除 | `.doc(id).remove()` | 仅 `data-cleanup` 云函数调用 |

> ⚠️ **搜索限制**：CloudBase NoSQL 不支持全文检索，MVP 阶段用标题前缀匹配，V2.5 接入向量搜索。

#### 2.2.6 `review_cards` — 复习卡片

| 操作 | SDK 方法 | 说明 |
|------|---------|------|
| **今日待复习** | `.where({ nextReview: db.command.lte(today) })` | ⭐ 最关键查询 |
| 某知识的所有卡片 | `.where({ knowledgeId })` | — |
| SM-2 更新 | `.doc(id).update({ mastery, interval, nextReview, lastQuality, reviewCount, lastReviewAt })` | 复习后更新 |
| 创建 | `.add({ question, answer, knowledgeId, ... })` | AI 生成或手动创建 |

**今日待复习查询完整示例**：

```javascript
// 加载今日应复习卡片（按掌握度升序，优先复习低掌握度）
const today = new Date()
today.setHours(0, 0, 0, 0)

const { data } = await db.collection('review_cards')
  .where({
    _openid: uid,
    nextReview: db.command.lte(today),   // <= 今天
    mastery: db.command.gt(0)             // 排除已失效（mastery=0）
  })
  .orderBy('mastery', 'asc')             // 低掌握度优先
  .orderBy('nextReview', 'asc')          // 早该复习的优先
  .limit(50)                             // 每次最多加载 50 张
  .get()
```

## 三、云函数接口

### 3.1 `ai-proxy` — AI 调用代理

> **作用**：避免 AI API Key 暴露前端，统一模型路由、费用统计、错误处理。

#### 3.1.1 调用方式

```javascript
// 前端调用（js/ai-chat.js）
const result = await app.callFunction({
  name: 'ai-proxy',
  data: {
    messages: [{ role: 'user', content: '你好' }],
    model: 'mimo',   // 可选，不传则自动路由
    stream: false     // MVP 不支持流式，V1.5 支持
  }
})

// result.result = { content: 'AI回复', tokens: 45, model: 'mimo-v2.5-pro' }
```

#### 3.1.2 入参规范

| 参数 | 类型 | 必填 | 说明 |
|------|:----:|:----:|------|
| `messages` | `Array<{role, content}>` | ✅ | OpenAI 格式消息历史 |
| `model` | `string` | ❌ | 模型代号（`mimo`/`deepseek`/`kimi`/`doubao`），不传则自动路由 |
| `stream` | `boolean` | ❌ | 是否流式返回，默认 `false` |
| `temperature` | `number` | ❌ | 创造性（0-1），默认 0.7 |
| `max_tokens` | `number` | ❌ | 最大输出 Token，默认 1000 |

#### 3.1.3 出参规范

**成功**：

```json
{
  "content": "AI 回复的完整文本",
  "tokens": 245,
  "model": "mimo-v2.5-pro",
  "cost": 0.00073
}
```

**失败**：

```json
{
  "error": "RATE_LIMITED",
  "message": "AI 服务繁忙，请稍后重试",
  "retryAfter": 30
}
```

#### 3.1.4 模型路由规则

| 任务类型 | 路由模型 | 切换条件 |
|---------|---------|---------|
| 中文问答、知识库检索 | MiMo（主力） | 默认 |
| 代码生成、调试 | DeepSeek-V3 | 用户手动切换 / 检测到代码块 |
| 长文写作、整理 | Kimi | 用户手动切换 / 检测到长文本 |
| 简单问答、快速响应 | 豆包 Lite | 月度预算 ≥ 80% 时自动切换 |

#### 3.1.5 SSE 流式格式（V1.5 支持）

```
data: {"content": "你", "tokens": 1}

data: {"content": "好，", "tokens": 2}

data: {"content": "我是", "tokens": 3}

data: [DONE]

X-Cost: 0.00073
X-Monthly-Cost: 2.45
```

### 3.2 `data-cleanup` — 数据清理定时任务

> **作用**：清理软删除数据、归档过期资讯、生成统计报告。

#### 3.2.1 触发方式

| 触发方式 | 说明 |
|---------|------|
| 定时触发 | 每天凌晨 2:00（CloudBase 定时触发器，cron: `0 2 * * *`） |
| 手动触发 | CloudBase 控制台 → 云函数 → 测试（用于调试） |

#### 3.2.2 清理策略

| 操作 | 条件 | 频率 |
|------|------|------|
| 永久删除知识条目 | `isDeleted: true` 且 `deletedAt` < 30 天前 | 每天 |
| 归档资讯 | `isSaved: true` 且 `createdAt` < 90 天前 | 每天 |
| 清理过期复习记录 | `reviewedAt` < 1 年前 | 每周 |
| 生成存储统计 | — | 每月 1 号 |

#### 3.2.3 出参（手动触发时返回）

```json
{
  "success": true,
  "cleaned": {
    "knowledgeItems": 3,
    "newsItems": 12,
    "reviewHistory": 45
  },
  "storage": {
    "totalDocs": 1234,
    "storageBytes": 5242880
  }
}
```

## 四、Auth 认证接口

### 4.1 匿名登录（V1 MVP）

```javascript
// js/auth.js

// 1. 检查登录态
async function checkLogin() {
  const loginState = await app.auth().getLoginState()
  if (loginState) {
    return loginState.uid   // 已登录，返回 uid
  }
  // 未登录 → 匿名登录
  return await anonymousLogin()
}

// 2. 匿名登录
async function anonymousLogin() {
  try {
    const result = await app.auth().anonymousAuthProvider().signIn()
    console.log('匿名登录成功，uid:', result.uid)
    return result.uid
  } catch (err) {
    console.error('登录失败', err)
    // 显示无法登录提示，引导刷新页面
    showToast('登录失败，请刷新页面重试', 'error')
  }
}

// 3. 登出
async function logout() {
  await app.auth().signOut()
  location.reload()
}
```

### 4.2 微信登录（V2）

```javascript
// V2 开放微信登录（需微信开放平台账号）
async function wechatLogin() {
  // 跳转到微信授权页（CloudBase 托管域名需配置到微信开放平台）
  await app.auth().weixinAuthProvider().signIn()
  // 回调后 CloudBase 自动处理，获取 uid
}
```

### 4.3 登录态持久化

CloudBase Auth SDK 自动将登录态存入 `localStorage`，有效期 30 天。无需应用层处理。

```javascript
// 页面加载时自动恢复登录态
app.auth().onLoginStateChanged(uid => {
  if (uid) {
    console.log('登录态恢复，uid:', uid)
    loadUserData(uid)
  } else {
    anonymousLogin()
  }
})
```

## 五、统一错误码

### 5.1 CloudBase SDK 错误码

| 错误码 | 说明 | 前端处理 |
|--------|------|---------|
| `INVALID_PARAM` | 参数错误（如 where 条件格式错误） | 检查入参，提示用户 |
| `PERMISSION_DENIED` | 安全规则拒绝（跨用户访问） | 重新登录，或提示权限不足 |
| `COLLECTION_NOT_EXIST` | 集合不存在（未初始化） | 引导用户等待初始化完成 |
| `QUOTA_EXCEEDED` | 免费额度用尽 | 提示用户升级或等待明天重置 |
| `NETWORK_ERROR` | 网络异常 | 提示检查网络，自动重试 2 次 |

### 5.2 云函数自定义错误码

| 错误码 | 说明 | 前端处理 |
|--------|------|---------|
| `RATE_LIMITED` | AI API 频率限制 | 等待 `retryAfter` 秒后重试 |
| `TIMEOUT` | 云函数/AI API 超时（15s） | 提示"服务繁忙"，建议拆分问题 |
| `API_KEY_INVALID` | AI API Key 失效/余额不足 | 提示联系管理员，切换备用模型 |
| `AI_ERROR` | AI API 返回错误（非 200） | 提示"AI 暂时不可用"，建议重试 |
| `MODEL_NOT_FOUND` | 请求的模型不存在 | 回退到默认模型（MiMo） |

### 5.3 HTTP 状态码映射（云函数返回）

| HTTP 状态码 | 含义 | 使用场景 |
|:-----------:|------|---------|
| 200 | 成功 | 正常返回 |
| 400 | 客户端错误 | 入参缺失/格式错误 |
| 429 | 频率限制 | AI API 限流 |
| 500 | 服务端错误 | 云函数崩溃/AI API 错误 |
| 504 | 网关超时 | 云函数执行超过 15s |

## 六、频率限制与预算控制

### 6.1 CloudBase 免费额度阈值

| 资源 | 免费额度 | 告警阈值 | 处理方式 |
|------|--------|:--------:|---------|
| 云数据库读 | 5 万次/天 | 4 万次 | Toast 提示"今日操作频繁" |
| 云数据库写 | 3 万次/天 | 2.4 万次 | 同上 |
| 云函数调用 | 1000 次/天 | 800 次 | 提示"AI 调用已达今日上限" |
| 静态托管流量 | 5 GB/月 | 4 GB | 提示用户 |
| 存储容量 | 2 GB | 1.6 GB | 提示清理数据 |

### 6.2 AI 月度预算控制

```
月度预算：¥5（可在云函数环境变量 AI_MONTHLY_BUDGET 配置）

控制逻辑（云函数 ai-proxy 内实现）：
  1. 每次调用后，将 cost 累加到 cloudbase 小计（可用云函数临时存储或独立集合 ai-monthly-cost）
  2. 累计 cost ≥ 预算 × 80% → 返回警告，前端 Toast 提示"本月 AI 预算已使用 80%"
  3. 累计 cost ≥ 预算 → 拒绝新请求，返回错误码 BUDGET_EXCEEDED
```

### 6.3 前端调用频率控制

```javascript
// js/ai-chat.js
let inflightCount = 0
const MAX_INFLIGHT = 2   // 同时最多 2 个 AI 请求

async function sendMessage(messages, model) {
  if (inflightCount >= MAX_INFLIGHT) {
    showToast('正在处理上一条消息，请稍候', 'warning')
    return
  }
  
  inflightCount++
  try {
    const result = await app.callFunction({
      name: 'ai-proxy',
      data: { messages, model }
    })
    // 处理回复...
  } finally {
    inflightCount--
  }
}
```
