## 一、接口体系概述

### 1.1 三层接口模型

本架构无传统RESTful接口，交互分为三层：

```
┌─────────────────────────────────────────────────────┐
│                  前端（浏览器）                         │
│                                                        │
│  js/auth.js  →  CloudBase Auth SDK  →  用户登录        │
│  js/db.js    →  CloudBase JS SDK   →  云数据库 CRUD    │
│  js/ai-chat.js → app.callFunction() → 云函数 ai-proxy  │
└──────────────────────┬────────────────────────────────┘
                       │ HTTPS（CloudBase 托管域名）
┌──────────────────────▼────────────────────────────────┐
│              CloudBase 云端                             │
│                                                         │
│  云数据库（NoSQL）← 安全规则（_openid 隔离）              │
│  云函数 ai-proxy ← 环境变量（AI API Key 不暴露前端）       │
│  CloudBase Auth ← 匿名登录 / 微信登录                    │
└─────────────────────────────────────────────────────────┘
```

## 二、CloudBase SDK 数据操作接口（来源：12-接口设计-v1.0）

### 2.1 通用调用模式

```javascript
import cloudbase from '@cloudbase/js-sdk'

const app = cloudbase.init({
  env: 'your-env-id',
  region: 'ap-shanghai'
})

const db = app.database()

async function queryCollection(collectionName, whereClause, orderClause) {
  try {
    const { data } = await db
      .collection(collectionName)
      .where(whereClause)
      .orderBy(orderClause.field, orderClause.direction)
      .get()
    return { success: true, data }
  } catch (err) {
    return { success: false, error: err }
  }
}
```

## 三、云函数接口（来源：12-接口设计-v1.0）

### 3.1 `ai-proxy` — AI 调用代理

> **作用**：避免 AI API Key 暴露前端，统一模型路由、费用统计、错误处理。

#### 3.1.1 调用方式

```javascript
const result = await app.callFunction({
  name: 'ai-proxy',
  data: {
    messages: [{ role: 'user', content: '你好' }],
    model: 'mimo',
    stream: false
  }
})
```

#### 3.1.2 入参规范

| 参数 | 类型 | 必填 | 说明 |
|------|:----:|:----:|------|
| `messages` | `Array<{role, content}>` | ✅ | OpenAI 格式消息历史 |
| `model` | `string` | ❌ | 模型代号（`mimo`/`deepseek`/`kimi`/`doubao`） |
| `stream` | `boolean` | ❌ | 是否流式返回，默认 `false` |
| `temperature` | `number` | ❌ | 创造性（0-1），默认 0.7 |
| `max_tokens` | `number` | ❌ | 最大输出 Token，默认 1000 |

#### 3.1.3 出参规范

**成功**：
```json
{
  "content": "AI 回复的完整文本",
  "tokens": 245,
  "model": "mimo-v2.5-pro",
  "cost": 0.00073
}
```

**失败**：
```json
{
  "error": "RATE_LIMITED",
  "message": "AI 服务繁忙，请稍后重试",
  "retryAfter": 30
}
```

#### 3.1.4 模型路由规则

| 任务类型 | 路由模型 | 切换条件 |
|---------|---------|---------|
| 中文问答、知识库检索 | MiMo（主力） | 默认 |
| 代码生成、调试 | DeepSeek-V3 | 用户手动切换 / 检测到代码块 |
| 长文写作、整理 | Kimi | 用户手动切换 / 检测到长文本 |
| 简单问答、快速响应 | 豆包 Lite | 月度预算 ≥ 80% 时自动切换 |

### 3.2 `data-cleanup` — 数据清理定时任务（接口编号：CF-001）

> **作用**：清理软删除数据、归档过期资讯、生成统计报告。

| 触发方式 | 说明 |
|---------|------|
| 定时触发 | 每天凌晨 2:00（cron: `0 2 * * *`） |
| 手动触发 | CloudBase 控制台测试 |

## 四、Auth 认证接口（来源：12-接口设计-v1.0）

### 4.1 匿名登录（V1 MVP）

```javascript
async function checkLogin() {
  const loginState = await app.auth().getLoginState()
  if (loginState) {
    return loginState.uid
  }
  return await anonymousLogin()
}

async function anonymousLogin() {
  const result = await app.auth().anonymousAuthProvider().signIn()
  return result.uid
}
```

### 4.2 微信登录（V2）

```javascript
async function wechatLogin() {
  await app.auth().weixinAuthProvider().signIn()
}
```

### 8.1 建议新增接口

| 接口编号 | 接口名称 | 用途 | 原因 |

### 8.2 建议合并接口

| 建议合并接口 | 当前接口 | 合并理由 | 优先级 |
|-------------|---------|---------|:------:|
| **DB-U-xxx 批量状态更新** | DB-U-002/003（暂停/恢复目标）、DB-U-006（完成任务）等 | 多个状态更新接口逻辑相似 | 中 |
| **DB-D-xxx 批量删除** | DB-D-005（批量删除知识条目）、DB-D-006（清空回收站）等 | 批量删除操作可统一接口 | 高 |

| **AGG-001** | 跨模块聚合复杂度高 | 考虑通过云函数实现 |

### 8.4 不建议合并的接口

| 接口类别 | 原因 |
|---------|------|

| **聚合查询接口（AGG-xxx）** | 每个聚合查询有不同的数据源和统计逻辑 |
| **核心CRUD接口** | 创建、读取、更新、删除操作语义明确，无需合并 |
