## 一、接口设计原则

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **单一职责** | 每个接口只处理单一模块的数据 |
| **模块隔离** | 接口按模块划分，便于独立开发和维护 |
| **RESTful 规范** | GET 查询、POST 创建、PUT 更新、DELETE 删除 |
| **AI 统一前缀** | 所有 AI 相关接口统一使用 `/api/ai/` 前缀 |
| **统计模式化** | 统计类接口统一模式，便于扩展 |

### 1.2 接口前缀规范

| 前缀 | 归属模块 | 说明 |
|------|----------|------|
| `/api/goals/*` | 学习计划 | 目标和里程碑相关 |
| `/api/tasks/*` | 学习计划 | 任务相关 |
| `/api/review/*` | 复习计划 | 复习卡片相关 |
| `/api/knowledge-items/*` | 知识库 | 知识条目相关 |
| `/api/categories/*` | 知识库 | 分类相关 |
| `/api/news/*` | 资讯 | 资讯相关 |
| `/api/chats/*` | AI对话 | 对话相关 |
| `/api/output-docs/*` | 知识沉淀 | 输出文档相关 |
| `/api/scraps/*` | 知识沉淀 | 灵感碎片相关 |
| `/api/user-settings/*` | 系统设置 | 用户设置相关 |
| `/api/ai/*` | AI通用 | AI暖身、快问快答等通用AI功能 |
| `/api/session/*` | 跨模块 | 跨模块聚合接口 |
| `/api/stats/*` | 统计中心 | 统一统计接口 |

## 十二、设计原则总结

### 12.1 不重不漏保证

| 原则 | 实现方式 |
|------|----------|
| **不重** | 相同功能只有一个接口定义；复用通过引用说明，非重复定义 |
| **不漏** | 每个原型功能点都有对应接口；跨模块引用明确标注来源 |
| **归属清晰** | 每个接口明确归属单一模块；跨模块接口标注调用来源 |
| **可扩展** | 统计接口采用模式化设计；新增模块只需添加新模块前缀 |

### 12.2 接口调用关系图

```
首页 (聚合层)
├── /api/stats/goals          ← 学习计划
├── /api/stats/review         ← 复习计划
├── /api/stats/news           ← 资讯
├── /api/stats/output-docs    ← 知识沉淀
├── /api/session/last         ← 跨模块
├── /api/review/yesterday-summary ← 复习计划
└── /api/ai/quiz              ← AI通用

知识库详情页
├── /api/ai/review/cards/generate  → 复习计划
└── /api/goals/from-knowledge      → 学习计划

资讯入库弹窗
├── /api/ai/news/{newsId}/import-suggestions  → AI通用
├── /api/categories/tree                       → 知识库
└── /api/goals                                 → 学习计划
```
