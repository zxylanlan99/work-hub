## 一、功能需求分析

### 1.1 核心功能点

| 功能模块 | 功能描述 | 对应原型区域 |
|---------|---------|-------------|
| **对话列表** | 展示所有历史对话，支持切换对话 | 左侧边栏 |
| **对话区域** | 展示当前对话消息，支持滚动查看历史 | 右侧主区域 |
| **消息发送** | 用户输入消息并发送给 AI | 底部输入框 |
| **模型选择** | 支持选择不同 AI 模型（MiMo/DeepSeek/Kimi/豆包） | 左侧边栏底部 |
| **对话操作** | 新建对话、导出对话、删除对话 | 顶部操作栏 |

### 1.2 页面结构

```
.ai-chat-page
    ├── .chat-container (双栏布局)
    │       ├── .chat-sidebar (左侧：对话列表 + 模型选择)
    │       │       ├── .sidebar-header
    │       │       │       ├── 标题 "对话列表"
    │       │       │       └── + 新对话按钮
    │       │       ├── .conversation-list
    │       │       │       └── .conversation-item (可点击切换)
    │       │       └── .model-selector
    │       │               ├── label "选择模型"
    │       │               └── .model-options
    │       │                       └── .model-option (可选择)
    │       │
    │       └── .chat-main (右侧：对话内容)
    │               ├── .chat-header
    │               │       ├── 标题 "AI助手"
    │               │       └── .header-actions (导出、删除)
    │               ├── .messages-container (消息列表，可滚动)
    │               │       └── .message (用户/助手消息交替)
    │               └── .chat-input-area
    │                       ├── textarea (消息输入)
    │                       └── .input-actions (上传文件、发送按钮)
```

## 二、布局与样式规范

### 2.1 布局结构

| 区域 | 宽度 | 样式特征 |
|------|------|---------|
| 左侧边栏 | 280px 固定 | 白色背景、圆角 12px、阴影 |
| 右侧主区域 | 自适应剩余空间 | 白色背景、圆角 12px、阴影 |
| 消息容器 | 100% | 最大高度 calc(100vh - 200px)、可滚动 |

| 元素 | CSS变量 | 说明 |

## 三、接口联调设计

### 3.1 数据集合操作

#### 3.1.1 `chats` — AI 对话集合

| 操作 | SDK 方法 | 接口编号 |
|------|---------|---------|
| **查询对话列表** | `db.collection('chats').orderBy('createdAt', 'desc').get()` | CHAT-001 |
| **创建新对话** | `db.collection('chats').add({ title, model })` | CHAT-002 |
| **更新对话标题** | `db.collection('chats').doc(id).update({ title })` | CHAT-003 |
| **删除对话** | `db.collection('chats').doc(id).remove()` | CHAT-004 |

#### 3.1.2 `messages` — 对话消息集合

| 操作 | SDK 方法 | 接口编号 |
|------|---------|---------|
| **查询对话消息** | `db.collection('messages').where({ chatId }).orderBy('createdAt', 'asc').get()` | MSG-001 |
| **发送用户消息** | `db.collection('messages').add({ role, content, chatId })` | MSG-002 |
| **写入 AI 回复** | `db.collection('messages').add({ role, content, model, tokens, cost, chatId })` | MSG-003 |
| **收藏消息** | `db.collection('messages').doc(id).update({ isStarred: true })` | MSG-004 |

### 3.2 云函数调用

#### 3.2.1 `ai-proxy` — AI 调用代理

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `messages` | `Array<{role, content}>` | ✅ | OpenAI 格式消息历史 |
| `model` | `string` | ❌ | 模型代号（mimo/deepseek/kimi/doubao） |
| `stream` | `boolean` | ❌ | 是否流式返回 |

**调用示例**：

```javascript
// 接口编号：AI-001
const result = await app.callFunction({
  name: 'ai-proxy',
  data: {
    messages: [{ role: 'user', content: '你好' }],
    model: 'mimo',
    stream: false
  }
})
// 返回：{ content, tokens, model, cost }
```

## 五、与技术规范一致性

|--------|---------|
| CSS 变量 | 使用 `--primary`、`--gray-*`、`--radius` 等全局变量 |
