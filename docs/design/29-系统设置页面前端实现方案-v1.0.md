## 一、功能需求分析

### 1.1 页面概述

系统设置页面是 StudyMind 的配置中心，用于管理用户个性化设置和系统配置。该页面采用卡片式布局，包含5个主要功能模块。

### 1.2 核心功能清单

| 功能模块 | 功能点 | 原型位置 |
|---------|--------|---------|
| **主题设置** | 界面主题切换（亮色/暗色/跟随系统） | 第125-145行 |
| **AI配置** | API Key管理（添加/编辑/删除，含状态标签）、默认模型、月度预算（只读展示）、预算进度条、预算提醒开关、超预算停止开关 | 第147-188行 |
| **通知设置** | 复习提醒、截止提醒、系统通知开关、提醒时间、免打扰时段（开始/结束时间） | 第190-219行 |
| **数据管理** | 导出数据、导入数据、清空所有数据（危险区域） | 第221-240行 |
| **自动备份** | 备份频率、云存储目标、手动备份 | 第242-267行 |

### 1.3 交互特性

| 特性 | 说明 |
|------|------|
| 自动保存 | 设置修改后即时生效，自动保存到云端 |
| 危险操作确认 | 清空数据等危险操作需要二次确认 |
| Toast提示 | 所有操作反馈使用统一Toast系统 |

### 4.1 接口清单

| 接口编号 | 接口名称 | 功能描述 | 调用方式 |
|---------|---------|---------|---------|
| **DB-R-034** | 获取用户设置 | 获取所有系统设置 | `settings.doc('user-settings').get()` |
| **DB-U-039** | 更新用户设置 | 保存设置变更 | `settings.doc('user-settings').update({...})` |
| **DB-R-035** | 获取API Key列表 | 获取已配置的API Key | `api_keys.orderBy('createdAt','desc').get()` |
| **DB-W-014** | 添加API Key | 新增API Key | `api_keys.add({provider, key, enabled})` |
| **DB-U-040** | 更新API Key | 修改API Key状态或内容 | `api_keys.doc(keyId).update({...})` |
| **DB-D-011** | 删除API Key | 删除指定API Key | `api_keys.doc(keyId).remove()` |
| **AGG-006** | 统计API调用量 | 获取月度调用统计 | 聚合查询 |
| **FUNC-001** | 导出数据 | 导出所有数据为JSON | `app.callFunction('export-data')` |
| **FUNC-002** | 导入数据 | 从JSON文件导入数据 | `app.callFunction('import-data')` |
| **FUNC-003** | 备份数据 | 执行数据备份 | `app.callFunction('backup-data')` |
| **FUNC-004** | 清空数据 | 清空所有用户数据 | `app.callFunction('clear-data')` |

### 4.2 数据模型

#### 4.2.1 `settings` 集合

| 字段名 | 类型 | 含义 | 默认值 |
|-------|------|------|--------|
| `theme` | string | 主题模式 | light/dark/system |
| `defaultModel` | string | 默认AI模型 | DeepSeek |
| `monthlyBudget` | number | 月度预算（元） | 20 |
| `budgetAlert` | boolean | 预算提醒开关 | true |
| `budgetStop` | boolean | 超预算停止开关 | true |
| `notificationReview` | boolean | 复习提醒开关 | true |
| `notificationDeadline` | boolean | 截止提醒开关 | true |
| `notificationSystem` | boolean | 系统通知开关 | true |
| `notificationTime` | string | 提醒时间 | 09:00 |
| `doNotDisturbStart` | string | 免打扰开始 | 22:00 |
| `doNotDisturbEnd` | string | 免打扰结束 | 08:00 |
| `backupFrequency` | string | 备份频率 | daily/weekly/off |
| `backupTarget` | string | 备份存储位置 | weiyun/cos/local |

#### 4.2.2 `api_keys` 集合

| 字段名 | 类型 | 含义 | 默认值 |
|-------|------|------|--------|
| `provider` | string | 服务商名称 | - |
| `key` | string | API Key（加密存储） | - |
| `enabled` | boolean | 是否启用 | true |
| `createdAt` | timestamp | 创建时间 | 自动 |

## 五、交互设计

### 5.1 设置更新流程

```
用户修改设置 → 自动保存 → Toast提示 → 刷新UI状态
```

### 5.2 API Key管理流程

```
点击"+ 添加Key" → 弹窗输入 → 验证保存 → 列表刷新
点击"编辑" → 弹窗修改 → 保存更新
点击"删除" → 确认提示 → 删除并刷新列表
```

### 5.3 危险操作流程

```
点击"清空所有数据" → 二次确认 → 调用接口 → Toast提示结果
```

## 八、技术要点

### 8.1 自动保存机制

```javascript
function saveSetting(key, value) {
  // 延迟保存，避免频繁调用
  settings[key] = value;
  window.toast('设置已保存', 'success');
  
  // 异步保存到数据库
  setTimeout(() => {
    // 调用DB-U-039接口
    saveSettingsToDB(settings);
  }, 500);
}
```

### 8.2 Toast提示复用

复用项目现有Toast系统：

```javascript
window.toast('主题已切换', 'success');
```

### 8.3 危险操作确认

```javascript
function clearData() {
  if (confirm('确定清空所有数据？此操作不可恢复！')) {
    // 调用FUNC-004接口
    window.toast('数据已清空', 'danger');
  }
}
```

## 九、安全与性能

### 9.1 注意事项

| 风险点 | 说明 | 解决方案 |
|--------|------|---------|
| API Key泄露 | 显示时需要脱敏处理 | 只显示前6位和后4位 |
| 数据误删 | 清空操作不可逆 | 强制二次确认 + 导出提醒 |
| XSS攻击 | 设置值可能包含恶意脚本 | 数据存储前进行转义 |
| 备份数据安全 | 备份文件包含敏感数据 | 加密存储 |
