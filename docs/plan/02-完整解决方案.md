## 一、系统概述

### 1.1 设计目标

StudyMind 是一个面向个人用户的 All-in-One 学习管理工具，旨在解决以下核心问题：

- **碎片化时间管理**：根据用户可用时间智能推荐学习内容
- **学习计划落地**：动态目标拆解，弹性调整不崩盘
- **资讯有效沉淀**：从信息过载中提取价值，自动关联知识体系
- **AI工具统一**：一个入口使用多个大模型，智能路由+费用透明
- **知识有序管理**：三级分类+自动关联+过期检测
- **科学复习机制**：基于遗忘曲线的智能复习推送
- **学习输出闭环**：内置模板，强制输出，学→用→写

### 1.2 设计原则

| 原则 | 说明 | 现状 |
|------|------|------|
| **云端优先** | 数据通过 CloudBase NoSQL 统一存储，多设备即时同步 | ✅ 已切换 |
| **渐进增强** | 基础功能不依赖AI/云服务，高级功能逐步叠加 | ✅ 保持不变 |
| **闭环驱动** | 输入即触发后续行动，拒绝"收藏即终点" | ✅ 保持不变 |
| **陪伴而非强制** | 循序渐进，暖身启动，不制造焦虑 | ✅ 保持不变 |
| **收敛聚焦** | 主题锁定防止贪多，一次只深挖一个主题 | ✅ 保持不变 |

> 📝 变更记录：原"本地优先"原则已调整为"云端优先"，对应技术栈从 SQLite+PWA 切换为 CloudBase。

## 三、技术栈详解

> 📝 变更记录：技术栈已从 React+Vite+Express+Prisma+SQLite 切换为纯 HTML/CSS/JS + CloudBase。以下为当前方案。

### 3.1 前端技术选型（当前方案）

| 技术 | 用途 |
|------|------|
| HTML5 + CSS3 + 原生 JS (ES2022+) | 前端三件套，无框架依赖 |
| CSS Variables | 主题系统（统一白色主题） |
| CloudBase JS SDK | 数据库 / 认证 / 云函数调用 |
| Hash-based SPA Router | 客户端路由（纯 JS 实现） |
| marked.js | Markdown 渲染 |
| Chart.js | 图表可视化（进度/统计） |

**选型理由**：
- 纯 HTML/CSS/JS 零构建步骤，CloudBase 静态托管直接部署
- 无打包工具/框架依赖，维护成本极低
- CloudBase SDK 直接操作数据库，无需自建后端 API

### 3.2 后端技术选型（当前方案）

| 技术 | 用途 |
|------|------|
| CloudBase 云函数 (Node.js 18) | AI 代理层（ai-proxy） |
| CloudBase 云数据库 (NoSQL) | 13 个集合，文档型存储 |
| CloudBase Auth | 用户认证（匿名/微信登录） |
| CloudBase 静态托管 | 前端部署 |
| 小米 MiMo API | 主 AI 模型（learning/general/writing） |
| SiliconFlow API | DeepSeek + Kimi（coding/reasoning） |
| 火山引擎 API | 豆包（fast/simple/translation） |

**选型理由**：
- CloudBase 无需管理服务器，按量付费起步成本极低
- 云函数代理 AI 调用，API Key 不暴露前端
- NoSQL 文档模型灵活，无需 Schema 迁移

### 3.3 数据存储方案（当前方案）

| 数据类型 | 存储方式 | 理由 |
|----------|----------|------|
| 业务数据 | CloudBase NoSQL（13集合） | 文档型存储，前端直接读写 |
| 用户认证 | CloudBase Auth | 内置微信/匿名登录 |
| 文件存储 | ~~不支持~~（明确排除附件上传） | 保持轻量化 |
| 全文搜索 | CloudBase 数据库正则 + 复合索引 | 满足 MVP 需求 |
| 向量搜索(V2.0) | CloudBase + 外部向量服务 | 后续迭代评估 |
| AI Key 等机密 | 云函数环境变量 | 不暴露前端 |

### 4.2 Prisma Schema 核心定义

```prisma
// schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"
  url      = env("DATABASE_URL")
}

model User {
  id            String    @id @default(uuid())
  name          String?
  email         String?   @unique
  avatar        String?
  preferences   String?   // JSON: {theme, defaultModel, notifications...}
  createdAt     DateTime  @default(now())
  updatedAt     DateTime  @updatedAt

  goals         LearningGoal[]
  knowledgeItems KnowledgeItem[]
  chatSessions  ChatSession[]
  reviewCards   ReviewCard[]
  outputDocs    OutputDoc[]
  dailyLogs     DailyLog[]
  categories    Category[]
  tags          Tag[]
}

model LearningGoal {
  id          String   @id @default(uuid())
  userId      String
  user        User     @relation(fields: [userId], references: [id])
  title       String
  description String?
  status      String   @default("active") // active, paused, completed, archived
  startDate   DateTime @default(now())
  targetDate  DateTime?
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  milestones  Milestone[]
}

model Milestone {
  id          String   @id @default(uuid())
  goalId      String
  goal        LearningGoal @relation(fields: [goalId], references: [id])
  title       String
  description String?
  targetDate  DateTime?
  status      String   @default("pending") // pending, in_progress, completed
  order       Int      @default(0)

  tasks       Task[]
}

model Task {
  id          String   @id @default(uuid())
  milestoneId String
  milestone   Milestone @relation(fields: [milestoneId], references: [id])
  title       String
  description String?
  dueDate     DateTime?
  status      String   @default("pending") // pending, in_progress, completed
  priority    String   @default("medium") // low, medium, high
  estimatedMinutes Int?
  completedAt DateTime?
  createdAt   DateTime @default(now())
}

model Category {
  id        String   @id @default(uuid())
  userId    String
  user      User     @relation(fields: [userId], references: [id])
  parentId  String?  // 自关联，支持三级分类
  parent    Category? @relation("CategoryHierarchy", fields: [parentId], references: [id])
  children  Category[] @relation("CategoryHierarchy")
  name      String
  type      String   // domain, project, topic
  level     Int      // 1, 2, 3
  color     String?
  order     Int      @default(0)

  knowledgeItems KnowledgeItem[]
}

model KnowledgeItem {
  id          String   @id @default(uuid())
  userId      String
  user        User     @relation(fields: [userId], references: [id])
  categoryId  String
  category    Category @relation(fields: [categoryId], references: [id])
  title       String
  content     String   // Markdown content
  sourceType  String?  // manual, chat, import, rss
  sourceUrl   String?
  aiSummary   String?
  relevanceScore Float? // 0-1, AI评分
  isArchived  Boolean  @default(false)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  tags        KnowledgeTag[]
  reviewCards ReviewCard[]
  chatSessions ChatSession[]
  outputDocs  OutputDocKnowledge[]
}

model Tag {
  id        String   @id @default(uuid())
  userId    String
  user      User     @relation(fields: [userId], references: [id])
  name      String
  color     String   @default("#3b82f6")

  knowledgeItems KnowledgeTag[]
}

model KnowledgeTag {
  knowledgeId String
  knowledge   KnowledgeItem @relation(fields: [knowledgeId], references: [id])
  tagId       String
  tag         Tag @relation(fields: [tagId], references: [id])

  @@id([knowledgeId, tagId])
}

model ChatSession {
  id            String   @id @default(uuid())
  userId        String
  user          User     @relation(fields: [userId], references: [id])
  knowledgeId   String?  // 关联到知识主题
  knowledge     KnowledgeItem? @relation(fields: [knowledgeId], references: [id])
  title         String   @default("新对话")
  model         String   // deepseek, kimi, doubao
  totalTokens   Int      @default(0)
  totalCost     Float    @default(0) // 元
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  messages      ChatMessage[]
}

model ChatMessage {
  id          String   @id @default(uuid())
  sessionId   String
  session     ChatSession @relation(fields: [sessionId], references: [id])
  role        String   // user, assistant, system
  content     String
  tokens      Int?     // 消耗的token数
  model       String?  // 实际使用的模型
  createdAt   DateTime @default(now())
}

model ReviewCard {
  id            String   @id @default(uuid())
  userId        String
  user          User     @relation(fields: [userId], references: [id])
  knowledgeId   String
  knowledge     KnowledgeItem @relation(fields: [knowledgeId], references: [id])
  nextReviewDate DateTime @default(now())
  reviewCount   Int      @default(0)
  masteryLevel  String   @default("new") // new, learning, reviewing, mastered
  intervalDays  Int      @default(1) // 下次复习间隔
  createdAt     DateTime @default(now())
}

model OutputDoc {
  id            String   @id @default(uuid())
  userId        String
  user          User     @relation(fields: [userId], references: [id])
  title         String
  type          String   // note, article, speech, practice
  content       String   // Markdown
  status        String   @default("draft") // draft, published
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt

  knowledgeItems OutputDocKnowledge[]
}

model OutputDocKnowledge {
  outputDocId   String
  outputDoc     OutputDoc @relation(fields: [outputDocId], references: [id])
  knowledgeId   String
  knowledge     KnowledgeItem @relation(fields: [knowledgeId], references: [id])

  @@id([outputDocId, knowledgeId])
}

model DailyLog {
  id              String   @id @default(uuid())
  userId          String
  user            User     @relation(fields: [userId], references: [id])
  date            DateTime @unique
  studyMinutes    Int      @default(0)
  tasksCompleted  Int      @default(0)
  notes           String?
  mood            String?  // great, good, neutral, tired, bad
}
```

```typescript
// 费用计算（元/千tokens）
const Pricing = {
  'deepseek-chat': { input: 0.001, output: 0.002 },
  'deepseek-r1': { input: 0.001, output: 0.004 },
  'kimi-moonshot': { input:0.003, output: 0.003 },
  'doubao-lite': { input: 0.0003, output: 0.0006 },
  'doubao-pro': { input: 0.0008, output: 0.002 }
};

function calculateCost(model: string, inputTokens: number, outputTokens: number): number {
  const pricing = Pricing[model];
  if (!pricing) return 0;
  return (inputTokens * pricing.input + outputTokens * pricing.output) / 1000;
}
```

```typescript
// 基于SM-2算法的简化版
function calculateNextReview(card: ReviewCard, quality: number): ReviewCard {
  // quality: 0-5，0=完全不会，5=完美回忆
  const { reviewCount, intervalDays, masteryLevel } = card;

  let newInterval: number;
  let newMastery: string;

  if (quality < 3) {
    // 回忆失败，重置
    newInterval = 1;
    newMastery = 'learning';
  } else {
    // 回忆成功
    if (reviewCount === 0) {
      newInterval = 1;
    } else if (reviewCount === 1) {
      newInterval = 3;
    } else {
      newInterval = Math.round(intervalDays * 2.5); // 简化版
    }
    newMastery = reviewCount > 3 ? 'mastered' : 'reviewing';
  }

  const nextReviewDate = addDays(new Date(), newInterval);

  return {
    ...card,
    reviewCount: reviewCount + 1,
    intervalDays: newInterval,
    nextReviewDate,
    masteryLevel: newMastery
  };
}
```
