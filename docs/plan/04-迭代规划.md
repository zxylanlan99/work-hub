## 二、V1（MVP）阶段（第 1-2 月）

### 2.1 阶段目标

跑通"学习计划 → 知识库 → AI对话 → 知识输出"的核心闭环。技术栈：**纯 HTML/CSS/JS + CloudBase（NoSQL + 云函数 + Auth + 静态托管）**。

### 2.2 技术栈

| 层 | 技术 |
|----|------|
| 前端 | 纯 HTML5 + CSS3 + 原生 JS (ES2022+)，CSS Variables 白色主题 |
| 数据库 | CloudBase NoSQL（13 个集合） |
| 后端 | CloudBase 云函数（ai-proxy：MiMo + DeepSeek/Kimi + 豆包路由） |
| 认证 | CloudBase Auth（匿名登录为主） |
| 部署 | CloudBase 静态托管 |
| AI 主模型 | **MiMo**（小米平台） |

### 2.3 详细任务分解

#### Week 1-2：项目初始化 + CloudBase 环境搭建

| 任务 | 交付物 | 验收标准 |
|------|--------|----------|
| CloudBase 环境创建 | 环境 ID + 密钥 | 可登录控制台 |
| 数据库初始化 | 13 个集合 + 索引 + 安全规则 | 前端 SDK 可读写 |
| 云函数 ai-proxy 部署 | 4 个模型代理均可调用 | 通过 HTTP 触发可返回 AI 回复 |
| 基础 HTML 布局 | index.html + 侧边栏 + 路由 | 7 项导航可用 |
| CSS 变量体系 | variables.css | 白色主题统一 |

#### Week 3-4：学习计划模块

| 任务 | 交付物 | 验收标准 |
|------|--------|----------|
| 目标管理 CRUD | goals 集合读写 | 可创建/编辑/删除目标 |
| AI 辅助拆解 | 节点 2A（目标澄清）+ 2B（路径生成）| 输入目标自动拆解为里程碑 |
| 里程碑 + 任务管理 | 里程碑/任务列表页面 | 拖拽调整、标记完成 |
| 进度仪表盘 | 首页进度组件 | 显示总体/本周完成度 |

#### Week 5-6：知识库模块

| 任务 | 交付物 | 验收标准 |
|------|--------|----------|
| 三级分类树 | categories 集合 + 树形 UI | 可创建领域→项目→主题三级 |
| 知识条目 CRUD | knowledge_items 集合 | 支持 Markdown 编辑 |
| 全文检索 | CloudBase 正则查询 | 0.5 秒内返回结果 |
| 自动分类引导 | 节点 5A（强制分类引导）| 录入知识时自动建议分类 |

#### Week 7-8：AI 对话 + 知识沉淀 + 联调

| 任务 | 交付物 | 验收标准 |
|------|--------|----------|
| AI 对话界面 | 聊天 UI + 模型切换 | MiMo + DeepSeek/Kimi 可对话 |
| 对话历史 | chats + messages 集合 | 可查看、搜索历史 |
| 对话收藏到知识库 | 一键保存对话精华 | 自动关联标签 |
| Markdown 笔记 | output_docs 集合 | 可创建/编辑/查看笔记 |
| 首页仪表盘 | 汇总视图 | 显示学习进度+近期知识+快捷入口 |
| 全流程联调 | 端到端测试 | 目标→学习→记笔记→AI对话全流程通 |

### 2.4 V1 里程碑检查点

## 六、V3 工程化升级（第 9-12 月）

### 6.1 阶段目标

**条件触发**：仅当用户量增长到需要更好的工程化支持时启动。从纯 HTML/JS 迁移到 React + TypeScript 框架。

### 6.2 变更内容

| 变更 | 说明 | 风险 |
|------|------|------|
| 框架迁移 | 纯 HTML/JS → React 19 + TypeScript | 需完整重写前端 |
| SSR 支持 | Next.js / Remix 框架 | 增加服务端复杂度 |
| 组件化 | Web Components → React Components | UI 行为需一致 |
| 国际化 | i18n 支持 | 英文用户可用 |
| 后端容器化 | 云函数 → CloudBase 容器 | 如函数冷启动成为瓶颈 |

### 6.3 核心不变原则

- 数据库始终 CloudBase NoSQL（平滑迁移，不丢数据）
- API Key 始终由云函数代理，任何阶段都不暴露前端
- V3 的 React 重写不影响已有数据，只是前端框架变更

### 6.4 里程碑

- [ ] React + TypeScript 前端可部署
- [ ] 与 V2.5 功能等价
- [ ] 构建流程完善（CI/CD）
- [ ] 目标用户量：**无上限**

## 十、风险与应对

| 风险 | 影响版本 | 应对策略 |
|------|----------|----------|
| MiMo API 不稳定 | V1+ | 云函数内置 DeepSeek/Kimi 自动降级 |
| AI API 费用超预期 | V1+ | 云函数记 tokens + 月度预算告警 |
| CloudBase 免费额度不足 | V1.5+ | 提前评估付费套餐，按需升级 |
| 匿名登录迁移困难 | V2 | V1 即同时支持匿名和邮箱，平滑过渡 |
| 纯 HTML/JS 难以维护 | V2+ | V3 分阶段迁移至 React，不阻塞前期交付 |
| 多用户安全规则复杂 | V2 | V1 提前设计 _openid 体系，V2 直接启用 |
