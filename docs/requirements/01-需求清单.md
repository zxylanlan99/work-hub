## 一、需求概述

### 1.1 产品背景

用户面临学习过程中的系统性痛点：时间碎片化、计划难以坚持、资讯过载且无法沉淀、多平台AI工具混用效率低、知识库混乱、缺乏复习机制、学习难以转化为实践输出。需要一个 All-in-One 的个人学习中枢，将分散的学习行为统一到一个闭环系统中。

### 1.2 产品定位

**StudyMind（学思）** — 一个云端优先型 All-in-One 个人学习助手（基于腾讯 CloudBase），以"陪伴式渐进学习"为核心理念，帮助用户实现"学得进去、记得住、用得上、写得出来"的学习闭环。

> 📝 变更记录：原定位"本地优先型"已调整为"云端优先型"，对应技术栈从 React+SQLite+PWA 切换为纯 HTML/CSS/JS + CloudBase（NoSQL + 云函数 + 静态托管）。

### 1.3 目标用户

- 主要用户：希望系统化管理个人学习的职场人士
- 使用场景：晚上/周末的成块学习时间 + 工作间隙的碎片时间
- 技术能力：具备基础互联网工具使用能力

## 二、痛点清单（原始需求来源）

### 痛点1：学习时间碎片化

| 项目 | 内容 |
|------|------|
| **问题描述** | 只有晚上或周末有成块时间，平时时间零散 |
| **影响** | 难以启动深度学习，学习计划经常被打断 |
| **期望** | 系统能根据可用时间自动推荐适配的学习内容 |
| **优先级** | P0 |

### 痛点2：学习计划难定、易变

| 项目 | 内容 |
|------|------|
| **问题描述** | 不好确定目标，不好制定计划，且计划常改 |
| **影响** | 目标模糊导致行动涣散，频繁改动导致计划崩盘 |
| **期望** | 支持动态目标管理，弹性调整不崩盘 |
| **优先级** | P0 |

### 痛点3：资讯沉淀五大难题

| 子问题 | 描述 | 优先级 |
|--------|------|--------|
| 3.1 渠道混乱 | 不知道哪些信息渠道可靠 | P1 |
| 3.2 信息过载 | 信息大爆炸，不知道哪些有用 | P0 |
| 3.3 不成体系 | 有用信息无法成体系 | P0 |
| 3.4 无法转化 | 沉淀资讯无法转化为后续学习资料 | P0 |
| 3.5 收藏即遗忘 | 简单收藏后后续找不到 | P1 |

### 痛点4：AI对话多平台混用

| 项目 | 内容 |
|------|------|
| **问题描述** | 豆包、DeepSeek、Kimi、MiMo、QClaw、WorkBuddy混用 |
| **影响** | 不知道哪个模型擅长哪方面，性价比不清楚 |
| **期望** | 统一对话入口，智能推荐模型，费用透明 |
| **优先级** | P0 |

> 📝 变更记录：新增 MiMo 为主模型（2026-06-11 概要/详细设计确认）。

### 痛点5：知识库混乱

| 项目 | 内容 |

| **问题描述** | 学习/工作/生活文件混在一起，有过时资料 |
| **影响** | 查找困难，资料堆积不知如何利用 |
| **期望** | 清晰分类、过期检测、关联推荐 |
| **优先级** | P0 |

### 痛点6：缺乏复习机制

| 项目 | 内容 |
|------|------|
| **问题描述** | 学完就忘，知识点零散，无温故计划 |
| **影响** | 学习效果差，投入产出比低 |
| **期望** | 基于遗忘曲线自动生成复习计划 |
| **优先级** | P1 |

### 痛点7：知识难转化为实践

| 项目 | 内容 |
|------|------|
| **问题描述** | 偶尔写写东西，难将知识转化为实践可用 |
| **影响** | 学完即终点，无法形成能力沉淀 |
| **期望** | 内置输出模板，学完后强制输出闭环 |
| **优先级** | P0 |

## 三、功能需求清单

### 3.1 学习计划模块

#### FR-PL-001 目标管理
- **需求描述**：支持创建学习目标，支持模糊目标（如"掌握微服务架构"）和具体目标
- **验收标准**：用户可以输入目标，AI辅助拆解为可衡量的里程碑
- **优先级**：P0

#### FR-PL-002 计划拆解
- **需求描述**：自动将目标拆解为周任务、日任务
- **验收标准**：目标创建后，系统自动生成任务树，用户可手动调整
- **优先级**：P0

#### FR-PL-003 弹性调整
- **需求描述**：支持拖拽调整任务、延期不崩盘、自动重新排期
- **验收标准**：修改一个任务的截止日期，后续相关任务自动顺延
- **优先级**：P0

#### FR-PL-004 时间感知推荐
- **需求描述**：根据用户历史活跃时段，智能推荐当前适合的学习模式
- **验收标准**：系统能判断"现在适合5分钟碎片阅读还是2小时深度学习"
- **优先级**：P1

#### FR-PL-005 主题锁定
- **需求描述**：开启一个主题后，系统主动过滤低关联内容，强制完成当前主题闭环
- **验收标准**：主题锁定期间，首页只显示该主题相关内容
- **优先级**：P1

#### FR-PL-006 进度追踪
- **需求描述**：可视化展示学习进度，包括总体完成度、本周完成度
- **验收标准**：提供进度条、完成率统计、趋势图表
- **优先级**：P0

#### FR-PL-007 学习模式切换
- **需求描述**：支持碎片模式（5-15分钟）、深度模式（1-2小时）、暖身模式（3分钟启动）
- **验收标准**：用户可根据当前时间选择模式，系统推荐适配内容
- **优先级**：P1

### 3.3 AI对话模块

#### FR-AI-001 统一对话界面
- **需求描述**：一个入口与多个AI模型对话
- **验收标准**：界面内可切换不同模型进行对话
- **优先级**：P0

#### FR-AI-002 模型路由
- **需求描述**：根据任务类型自动推荐最优模型
- **验收标准**：输入问题后，系统分析并推荐"此问题建议用DeepSeek回答"
- **优先级**：P1

#### FR-AI-003 费用透明
- **需求描述**：每次对话前显示预估费用，提供月度统计
- **验收标准**：对话前显示"预估消耗：0.02元"，月度报表展示各模型费用
- **优先级**：P1

#### FR-AI-004 对话历史保存
- **需求描述**：所有对话自动保存，支持搜索和回顾
- **验收标准**：对话结束后自动归档，可按关键词搜索历史对话

#### FR-AI-005 对话即知识

- **验收标准**：对话中点击"收藏知识点"，自动提取并关联标签存入知识库
- **优先级**：P0

#### FR-AI-006 上下文绑定
- **需求描述**：对话可绑定到具体学习计划或知识主题
- **验收标准**：创建对话时选择"关联主题：微服务架构"，AI了解相关背景

#### FR-AI-007 支持模型列表
- **需求描述**：支持 MiMo（主）、DeepSeek、Kimi、豆包四个模型
- **验收标准**：四个模型均可通过云函数代理正常对话，MiMo 为默认模型
- **优先级**：P0
