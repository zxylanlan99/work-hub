Go 语言并发模型深度解析

Go 语言的并发模型基于 CSP（Communicating Sequential Processes）理论，通过 goroutine 和 channel 实现轻量级并发。这是 Go 语言最核心的设计之一。

一、核心概念

Go 语言的并发哲学可以用一句话概括："不要通过共享内存来通信，而要通过通信来共享内存。"

在 Go 中，我们不应该使用锁和共享变量来协调多个 goroutine，而应该使用 channel 进行数据传递和同步。这种设计大大简化了并发编程的复杂度。

Goroutine 是 Go 运行时管理的轻量级线程，初始栈仅 2KB，由 GMP 调度模型管理，可以在少量 OS 线程上运行大量 goroutine。

二、GMP 调度模型

GMP 是 Go 调度器的核心设计，分别代表：

  • G (Goroutine)：待执行的任务，包含栈、指令指针、状态等
  • M (Machine)：操作系统线程，负责实际执行代码
  • P (Processor)：逻辑处理器，维护本地可运行队列，数量默认等于 CPU 核数

三、Channel 原理

Channel 是 Go 语言中 goroutine 之间通信和同步的核心机制。其内部实现是一个环形缓冲区，配合发送/接收队列实现阻塞通信。

type hchan struct {
    qcount   uint           // 当前元素数量
    dataqsiz uint           // 环形队列大小
    buf      unsafe.Pointer // 环形队列指针
    elemsize uint16         // 元素大小
    closed   uint32         // 是否已关闭
    sendx    uint           // 发送索引
    recvx    uint           // 接收索引
    recvq    waitq          // 接收等待队列
    sendq    waitq          // 发送等待队列
    lock     mutex          // 互斥锁
}

四、实战案例

以下是一个使用 goroutine + channel 实现的并行文件处理：

func processFiles(files []string) {
    jobs := make(chan string, len(files))
    results := make(chan Result, len(files))
    for w := 1; w <= 3; w++ {
        go worker(w, jobs, results)
    }
    for _, file := range files {
        jobs <- file
    }
    close(jobs)
    for range files {
        fmt.Println(<-results)
    }
}
